**版本**: v2.0

## 一、问题现象

### 1.1 当前现象

在增量下载测试中，发现以下问题：

1. **范围查询覆盖已存在数据**：当缺失6个报告期时，使用范围查询策略，下载了10条记录（包含已存在的2条）
2. **去重只在存储层面生效**：通过 `Deduplication completed: input=10, output=8, removed=2` 可以看到重复数据被移除，但API请求层面没有避免重复下载
3. **Type C 接口超时**：从2005年开始生成锚点值，导致84个任务超时

### 1.2 具体日志表现

## 二、问题原因

### 2.1 根本原因

**`MAX_PRECISE_QUERIES` 阈值设置过低**：

1. **阈值与预期矛盾**：`MAX_PRECISE_QUERIES = 3`，但期望在缺失6个报告期时仍使用精确查询
2. **范围查询覆盖已存在数据**：范围查询 `20230101 ~ 20241231` 包含了已存在的 `20240331`
3. **下载层面没有去重**：虽然存储层面去除了2条重复，但API请求已经发出

### 2.2 代码层面的问题

#### 问题1：MAX_PRECISE_QUERIES 阈值过低

```python
# coverage_manager.py 第 927 行（当前代码）
MAX_PRECISE_QUERIES = 3  # 最多 3 个精确查询，超过则使用范围查询
```

当缺失6个报告期时，触发范围查询策略，导致下载了额外数据。

#### 问题2：Type C 接口历史范围无限制

```python
# _detect_date_anchor_gaps() 当前逻辑
anchor_values = self._generate_anchor_values(start_date, end_date, anchor_param)
```

当 `start_date` 为2005年时，会生成从2005年到现在的所有锚点值（约84个），导致任务超时。

## 四、改进方案

### 4.1 核心改进思路

1. **提高 `MAX_PRECISE_QUERIES` 阈值**：让更多情况使用精确查询，避免范围查询覆盖已存在数据
2. **限制 Type C 接口历史范围**：避免从2005年开始生成锚点，减少任务数量
3. **优化范围查询策略**：当必须使用范围查询时，使用分段范围查询避免覆盖已存在数据

```python
def _get_effective_start_date(
    self, 
    requested_start_date: str, 
    stock_info: Optional[Dict[str, Any]], 
    max_history_years: int = 5
) -> str:
    """
    计算有效的开始日期，限制历史范围
    
    Args:
        requested_start_date: 用户请求的开始日期
        stock_info: 股票信息（包含上市日期等）
        max_history_years: 最大回溯年数
        
    Returns:
        有效的开始日期
    """
    from datetime import datetime, timedelta
    
    # 计算最大回溯日期
    now = datetime.now()
    max_history_date = (now - timedelta(days=max_history_years*365)).strftime('%Y%m%d')
    
    # 取请求日期和最大回溯日期的较大值（更近的日期）
    effective_start = max(requested_start_date, max_history_date)
    
    # 如果有股票上市日期，且上市日期晚于effective_start，使用上市日期
    if stock_info:
        list_date = stock_info.get('list_date')
        if list_date and list_date > effective_start:
            effective_start = list_date
    
    return effective_start
```

**理由**：
- 避免从2005年开始生成84个锚点值导致超时
- 5年历史范围足够满足大部分分析需求
- 如果股票上市时间晚于5年前，使用上市日期作为开始日期

#### 改进3：实现分段范围查询（可选优化）

**文件**: `/home/quan/testdata/aspipe_v4/app4/core/coverage_manager.py`

当缺失报告期数量确实很多时，使用分段范围查询避免覆盖已存在数据：

```python
def _build_segmented_range_queries(
    self, 
    ts_code: str, 
    missing_periods: List[str]
) -> List[Dict[str, Any]]:
    """
    将缺失的报告期分段，生成多个不重叠的范围查询
    
    例如：缺失 ['20230331', '20230630', '20230930', '20240630', '20240930', '20241231']
    已存在：['20240331']
    
    分段结果：
    - 段1: ['20230331', '20230630', '20230930'] -> 20230101 ~ 20230930
    - 段2: ['20240630', '20240930', '20241231'] -> 20240401 ~ 20241231
    
    避免范围查询覆盖已存在的 20240331
    """
    if not missing_periods:
        return []
    
    sorted_periods = sorted(missing_periods)
    segments = []
    current_segment = [sorted_periods[0]]
    
    for i in range(1, len(sorted_periods)):
        prev = sorted_periods[i-1]
        curr = sorted_periods[i]
        
        # 如果相邻报告期在同一年且相邻季度，合并
        if self._are_adjacent_quarters(prev, curr):
            current_segment.append(curr)
        else:
            # 不连续，保存当前段，开始新段
            segments.append(current_segment)
            current_segment = [curr]
    
    segments.append(current_segment)
    
    # 为每个段生成范围查询
    queries = []
    for segment in segments:
        queries.append({
            'ts_code': ts_code,
            'start_date': self._quarter_start(segment[0]),
            'end_date': segment[-1]
        })
    
    return queries

def _are_adjacent_quarters(self, period1: str, period2: str) -> bool:
    """判断两个报告期是否相邻（同一年内的相邻季度）"""
    year1, q1 = int(period1[:4]), int(period1[4:6]) // 3
    year2, q2 = int(period2[:4]), int(period2[4:6]) // 3
    
    # 同一年内，季度差为1
    if year1 == year2:
        return abs(q1 - q2) == 1
    
    # 跨一年，前一年Q4和下一年Q1
    if year2 == year1 + 1:
        return q1 == 4 and q2 == 1
    
    return False

def _quarter_start(self, period: str) -> str:
    """获取报告期对应的季度开始日期"""
    year = period[:4]
    month_day = period[4:]
    quarter_start_map = {
        '0331': f'{year}0101',
        '0630': f'{year}0401',
        '0930': f'{year}0701',
        '1231': f'{year}1001',
    }
    return quarter_start_map.get(month_day, f'{year}0101')
```

**使用方式**：

```python
# 在 _detect_report_period_gaps() 中
if len(missing_periods) <= MAX_PRECISE_QUERIES:
    # 使用精确查询
    return self._build_precise_queries(ts_code, missing_periods)
else:
    # 使用分段范围查询，避免覆盖已存在数据
    return self._build_segmented_range_queries(ts_code, missing_periods)
```

本方案针对4种接口类型（A/B/C/D）在8种输入组合下的下载去重问题，提出了精简而有效的改进方案：

   - 解决任务超时问题
   - 5年范围满足大部分分析需求

3. **分段范围查询**（可选优化）
   - 当缺失报告期确实很多时使用
   - 避免范围查询覆盖已存在数据
