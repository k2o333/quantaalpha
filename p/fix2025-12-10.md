# 数据下载问题修复方案 (2025-12-10)

## 问题概述

根据日志分析和接口文档查阅，当前数据下载存在以下主要问题：

1. **参数缺失问题**：
   - `cyq_chips` 接口缺少必需参数 `ts_code`
   - `broker_recommend` 接口缺少必需参数 `month`

2. **分页限制问题**：
   - 多个接口存在单次请求返回数据量限制，需要分页或循环提取完整数据
   - 部分接口默认只返回第一页数据，导致数据不完整

3. **接口失效问题**：
   - `moneyflow_ths`、`moneyflow_cnt_ths`、`moneyflow_ind_ths`、`forecast`、`express` 等接口返回0条记录

## 解决方案

### 1. 修复参数缺失问题

#### cyq_chips 接口
- **问题**：调用时未传递 `ts_code` 参数，而是传递了错误的 `trade_date` 参数
- **修复方法**：
  ```python
  # 错误示例
  pro.cyq_chips(trade_date='20231101')

  # 正确示例
  pro.cyq_chips(ts_code='000001.SZ', start_date='20231101', end_date='20231130')
  ```
  或者使用单个交易日
  ```python
  pro.cyq_chips(ts_code='000001.SZ', trade_date='20231101')
  ```

#### broker_recommend 接口
- **问题**：调用时传递了错误的 `start_date` 和 `end_date` 参数，而非必需的 `month` 参数
- **修复方法**：
  ```python
  # 错误示例
  pro.broker_recommend(start_date='20230101', end_date='20231231')

  # 正确示例
  pro.broker_recommend(month='202311')
  ```

### 2. 实现分页下载机制

#### 对于有分页限制的接口，应实现循环提取：

```python
def download_with_pagination(api_func, **kwargs):
    """
    分页下载数据的通用函数
    """
    all_data = []
    offset = 0
    limit = 2000  # 默认为cyq_chips限制，根据具体接口调整

    while True:
        # 添加分页参数
        kwargs['offset'] = offset
        kwargs['limit'] = limit

        # 调用API
        data = api_func(**kwargs)

        if data is None or len(data) == 0:
            break

        all_data.extend(data)

        # 如果返回数据少于限制数量，说明已到最后一页
        if len(data) < limit:
            break

        offset += limit

    return all_data
```

#### 针对具体接口的分页参数设置：
- `cyq_chips`: 单次最大2000条，可以按股票代码和日期循环提取
  ```python
  # 对于单个股票的长期数据
  df = pro.cyq_chips(ts_code='000001.SZ', start_date='20230101', end_date='20231231', limit=2000, offset=0)
  ```
- `broker_recommend`: 单次最大1000行数据，可按月份循环提取
  ```python
  # 按月份循环
  months = ['202301', '202302', '202303', ...]
  all_data = []
  for month in months:
      df = pro.broker_recommend(month=month)
      all_data.append(df)
  ```
- `moneyflow_dc`: 单次最大获取6000条数据，可根据日期或股票代码循环提取
  ```python
  # 对于大范围日期数据
  df = pro.moneyflow_dc(trade_date='20231101', limit=6000, offset=0)
  ```
- `stk_factor`: 单次最大10000条，可以循环或者分页提取
  ```python
  # 分页提取
  df = pro.stk_factor(trade_date='20231101', limit=10000, offset=0)
  ```
- `cyq_perf`: 单次最大5000条，可以分页或者循环提取
  ```python
  # 分页提取
  df = pro.cyq_perf(trade_date='20231101', limit=5000, offset=0)
  ```
- `report_rc`: 单次最大3000条，可分页和循环提取所有数据
  ```python
  # 分页提取
  df = pro.report_rc(period='20231231', limit=3000, offset=0)
  ```

### 3. 替换失效接口或添加异常处理

对于返回0条记录的接口，建议：

1. **确认接口状态**：
   - 检查是否需要更高权限（如5000积分）
   - 确认接口是否已废弃

2. **添加异常处理**：
   ```python
   def safe_download(api_func, **kwargs):
       try:
           data = api_func(**kwargs)
           if data is None or len(data) == 0:
               logger.warning(f"接口 {api_func.__name__} 返回空数据")
               return None
           return data
       except Exception as e:
           logger.error(f"接口 {api_func.__name__} 调用失败: {e}")
           return None
   ```

### 4. 实现智能分页下载机制

为有分页限制的接口实现自动分页下载：

```python
def download_with_pagination_control(api_func, limit_per_call, **base_kwargs):
    """
    根据接口的限制自动分页下载数据
    """
    all_data = []
    offset = 0

    while True:
        # 添加分页参数
        kwargs = base_kwargs.copy()
        kwargs['offset'] = offset
        kwargs['limit'] = limit_per_call

        try:
            data = api_func(**kwargs)
        except Exception as e:
            logger.error(f"分页下载失败, offset={offset}: {e}")
            break

        if data is None or len(data) == 0:
            break

        all_data.extend(data)

        # 如果返回数据少于限制数量，说明已到最后一页
        if len(data) < limit_per_call:
            break

        offset += limit_per_call

    return all_data
```
