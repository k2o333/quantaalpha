| 依赖项 | 状态 | 位置 |

## 二、设计决策（已确认）

**用户确认的处理逻辑**：
- 如果用户指定了 `--start_date`，则使用用户指定的日期
- 如果用户未指定 `--start_date`，则使用股票上市日期

---

## 三、推荐的代码修改

```python
# 在文件顶部添加导入
from core.constants import DEFAULT_STOCK_START_DATE

# 修改 _update_with_stock_gap_detection 方法
# 第 568 行附近

# --update 模式：始终进行全量缺口检测
user_provided_dates = True  # 强制全量检测

for stock in stock_list:
    ts_code = stock.get('ts_code')

    # 确定起始日期：优先使用用户指定的日期，否则使用股票上市日
    if options.start_date:
        stock_start_date = options.start_date
    else:
        stock_start_date = stock.get('list_date') or DEFAULT_STOCK_START_DATE

    # 检测该股票的数据缺口
    gap_tasks = self.coverage_manager.detect_stock_gaps(
        interface_name=interface_name,
        ts_code=ts_code,
        start_date=stock_start_date,  # 使用计算后的起始日期
        end_date=options.end_date or date_range.end_date,  # 同样考虑用户指定的结束日期
        interface_config=interface_config,
        user_provided_dates=user_provided_dates,  # True
        stock_info=stock
    )
```

## 四、潜在风险

### 1. 性能考虑

对于上市时间较早的股票（如 1991 年上市的平安银行），全量缺口检测会扫描 35 年的交易日历数据。

### 2. 影响范围

修改后影响所有 Type A 接口的 `--update` 模式行为：

| 接口 | 数据日期字段 |
|------|-------------|
| `cyq_chips` | `trade_date` |
| `moneyflow_dc` | `trade_date` |
| `stk_factor_pro` | `trade_date` |
