### 评估结论 2：downloader层存在过度复杂的跳过逻辑 ✅ 正确，但建议保留而非简化

**代码验证**：

`app4/core/downloader.py:493-509` 确实存在多层判断：

```python
skip_gap_detection = False
if context and context.skip_stock_level_detection:
    skip_gap_detection = True
elif context and context.gap_params:
    skip_gap_detection = True
else:
    # 检查参数中是否已包含日期锚点值
    param_defs = interface_config.get("parameters", {})
    for param_name in stock_params:
        param_def = param_defs.get(param_name, {})
        if param_def.get("is_date_anchor", False) and stock_params.get(param_name):
            skip_gap_detection = True
            break
```

**评估**：✅ 正确，但建议**保留而非简化**。

**原因分析**：

1. `context.skip_stock_level_detection` - 用于标记在 params_builder 层已完成检测（当前未使用）
2. `context.gap_params` - 用于存储 params_builder 层生成的缺口参数（当前未使用）
3. **日期锚点参数检查** - 这是实际生效的逻辑，用于检测参数中是否已包含日期锚点值

**为什么不能简化为 `skip_gap_detection = True`**：

- 该逻辑在 `download_single_stock` 方法中，用于 **stock_loop 模式**
- 对于 stock_loop 模式，如果参数中已包含日期锚点值（如 `ann_date`、`period`），说明参数是由 coverage_manager 缺口检测生成的，不应重复检测
- 对于其他情况（参数只有 `start_date`/`end_date`），需要执行缺口检测

**结论**：该逻辑是必要的，但可以优化代码结构使其更清晰。

## 发现的问题：reverse_date_range + is_date_anchor 模式的增量下载缺失

### 问题描述

虽然 `reverse_date_range` 模式的参数转换逻辑已实现，但**缺少增量下载（跳过已下载日期）的功能**。

### 当前行为

```
用户命令: python app4/main.py --update --interface cyq_perf --start_date 20260205 --end_date 20260220

执行流程:
1. pagination._apply_date_anchor_range() 生成参数:
   [{trade_date: "20260220"}, {trade_date: "20260219"}, ..., {trade_date: "20260205"}]
   
2. pagination_executor._execute_sequential() 执行请求
   - 每个请求都会被发送，即使数据已存在
   - 没有覆盖率检查！
```

### 问题定位

`pagination_executor.py:122-148` 中的 `_execute_sequential` 方法：

```python
for idx, params in enumerate(params_list):
    if progress_callback:
        progress_callback(idx + 1, len(params_list))
    
    if coverage_manager:
        if self._should_skip_by_coverage(interface_config, params, coverage_manager):
            continue
    
    data = self._execute_single_request(...)
```

**问题**：`_should_skip_by_coverage` 方法使用 `strategy='default'`，没有针对 `is_date_anchor` 参数使用 `period` 策略。

**验证代码** (`pagination_executor.py:217-238`)：

```python
def _should_skip_by_coverage(self, interface_config: Dict[str, Any], params: Dict[str, Any], 
                            coverage_manager: Any) -> bool:
    # ...
    if '_time_window' in params:
        strategy = 'date_range'
    elif '_period_query' in params:  # period_range 模式
        strategy = 'period'
    elif '_stock_info' in params:
        strategy = 'stock'
    elif '_type_value' in params:
        strategy = 'type'
    else:
        strategy = 'default'  # ← reverse_date_range + is_date_anchor 走这里
```

虽然后续有处理日期锚定的逻辑：

```python
if date_anchor_param:
    if date_anchor_param in clean_params:
        return coverage_manager.should_skip(api_name, clean_params, strategy='period')
```

但这使用了 `period` 策略，而 `period` 策略是针对报告期检测的，不适用于 `trade_date` 参数。

## 补充方案：reverse_date_range + is_date_anchor 增量下载

### 目标

为 `reverse_date_range + is_date_anchor` 模式添加增量下载支持，跳过已下载的日期。

### 修改方案

#### 修改点 1: coverage_manager.py - 新增日期锚点跳过检测

**位置**：`app4/core/coverage_manager.py` 中添加新方法

```python
def should_skip_date_anchor(
    self,
    interface_name: str,
    anchor_param: str,
    anchor_value: str
) -> bool:
    """
    检查日期锚点值是否已存在（用于 reverse_date_range + is_date_anchor 模式）
    
    Args:
        interface_name: 接口名称
        anchor_param: 日期锚定参数名，如 'trade_date'
        anchor_value: 日期锚定值，如 '20260205'
        
    Returns:
        True 表示已存在（应跳过），False 表示不存在（应下载）
    """
    try:
        interface_config = self.config_loader.get_interface_config(interface_name)
        detection_config = interface_config.get("duplicate_detection", {})
        date_column = detection_config.get("date_column", anchor_param)
        
        cache_key = f"{interface_name}:anchor:{date_column}"
        
        with self._cache_lock:
            if cache_key not in self._cache:
                # 加载所有已有日期
                df = self.storage_manager.read_interface_data(
                    interface_name, columns=[date_column]
                )
                
                if df.is_empty():
                    self._cache[cache_key] = set()
                else:
                    dates = set()
                    for date_val in df[date_column]:
                        formatted = format_date(date_val)
                        if formatted:
                            dates.add(formatted)
                    self._cache[cache_key] = dates
                    logger.info(
                        f"[{interface_name}] 已加载 {len(dates)} 个已有日期锚点值"
                    )
            
            return anchor_value in self._cache[cache_key]
            
    except Exception as e:
        logger.warning(f"日期锚点检测失败 {interface_name}/{anchor_value}: {e}")
        return False
```

#### 修改点 2: pagination_executor.py - 使用新的跳过检测

**位置**：`app4/core/pagination_executor.py` 的 `_should_skip_by_coverage` 方法

```python
def _should_skip_by_coverage(self, interface_config: Dict[str, Any], params: Dict[str, Any], 
                            coverage_manager: Any) -> bool:
    """根据覆盖率判断是否跳过请求"""
    api_name = interface_config.get('api_name', '')
    
    # 检测日期锚定参数（适用于 reverse_date_range + is_date_anchor 模式）
    param_defs = interface_config.get('parameters', {})
    for param_name, param_def in param_defs.items():
        if param_def.get('is_date_anchor', False) and param_name in params:
            # 使用日期锚点跳过检测
            anchor_value = params.get(param_name)
            if anchor_value:
                return coverage_manager.should_skip_date_anchor(
                    api_name, param_name, anchor_value
                )
            return False
    
    # 原有逻辑...
    if '_time_window' in params:
        strategy = 'date_range'
    elif '_period_query' in params:
        strategy = 'period'
    elif '_stock_info' in params:
        strategy = 'stock'
    elif '_type_value' in params:
        strategy = 'type'
    else:
        strategy = 'default'
    
    clean_params = {k: v for k, v in params.items() if not k.startswith('_')}
    try:
        return coverage_manager.should_skip(api_name, clean_params, strategy=strategy)
    except:
        return False
```
