# TuShare接口参数优化下载方案

## 概述

本方案旨在通过分析TuShare接口特点，优化下载参数，实现更精确的数据下载，避免重复下载和无效请求。

## 核心思路

1. **利用交易日历**：根据交易日历计算实际需要下载的日期
2. **参数预计算**：根据接口特点和API限制，预先计算最优的下载参数
3. **增量更新**：基于本地已存储数据的主键，只下载缺失的数据

## 接口参数计算策略

### 1. 日线数据接口 (daily)

#### 参数特点
- 输入参数：`ts_code`, `start_date`, `end_date`, `trade_date`
- 输出主键：`(ts_code, trade_date)`
- API限制：单次最大6000条数据

#### 计算策略
```python
def calculate_daily_download_params(ts_code, start_date, end_date):
    """
    计算日线数据的最优下载参数
    """
    # 1. 获取交易日历
    trading_days = get_trading_calendar(start_date, end_date)
    
    # 2. 按股票和日期范围生成参数
    if ts_code:
        # 单股票下载：按日期窗口分批
        batch_size = 3000  # 预留空间给API波动
        params_list = []
        for i in range(0, len(trading_days), batch_size):
            batch_days = trading_days[i:i+batch_size]
            params_list.append({
                'ts_code': ts_code,
                'start_date': batch_days[0],
                'end_date': batch_days[-1]
            })
    else:
        # 全市场下载：按日期分批
        params_list = []
        for trade_date in trading_days:
            params_list.append({
                'trade_date': trade_date
            })
    
    return params_list
```

def calculate_finance_download_params(ts_code, period_range):

## 增量更新策略

### 1. 主键检查机制
```python
def should_download_record(interface_name, record_data):
    """
    检查记录是否已存在，决定是否下载
    """
    primary_keys = get_primary_keys(interface_name, record_data)
    return not record_exists(interface_name, primary_keys)
```

### 2. 数据去重流程
```python
def download_with_dedup(interface_name, params):
    """
    带去重的数据下载流程
    """
    # 1. 获取原始数据
    raw_data = api_call(interface_name, params)
    
    # 2. 过滤已存在的记录
    new_data = []
    for record in raw_data:
        if should_download_record(interface_name, record):
            new_data.append(record)
    
    return new_data
```
