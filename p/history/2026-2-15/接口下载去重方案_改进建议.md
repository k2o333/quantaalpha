## 一、原方案问题分析

### 1.1 MAX_PRECISE_QUERIES 阈值与预期矛盾

原方案代码（第 135 行）：
```python

```

原方案预期日志（第 182 行）：
```
[000001.SZ] 使用精确查询策略（6 个缺失报告期）
```

**矛盾**：当缺失 6 个报告期时，`6 > 3` 会触发范围查询，而非精确查询。

**测试证据**（income_vip_2_output.txt）：
```

Downloaded 10 records
Deduplication completed: input=10, output=8, removed=2
```

**结论**：范围查询覆盖了已存在的数据（20240331），导致下载了 10 条而非 6 条。

### 2.2 范围查询时排除已存在数据

如果必须使用范围查询，应该只查询缺失的日期范围，而非整个范围：

**当前问题**：
```
缺失: ['20230331', '20230630', '20230930', '20240630', '20240930', '20241231']
范围查询: 20230101 ~ 20241231  # 包含已存在的 20240331
```

**改进方案 - 分段范围查询**：
```python
def _build_segmented_range_queries(self, ts_code: str, missing_periods: List[str]) -> List[Dict]:
    """将缺失的报告期分段，生成多个不重叠的范围查询"""
    if not missing_periods:
        return []
    
    sorted_periods = sorted(missing_periods)
    segments = []
    current_segment = [sorted_periods[0]]
    
    for i in range(1, len(sorted_periods)):
        # 如果相邻报告期在同一年且相邻季度，合并
        prev = sorted_periods[i-1]
        curr = sorted_periods[i]
        if self._are_adjacent_quarters(prev, curr):
            current_segment.append(curr)
        else:
            segments.append(current_segment)
            current_segment = [curr]
    segments.append(current_segment)
    
    # 为每个段生成范围查询
    queries = []
    for segment in segments:
        queries.append({
            'ts_code': ts_code,
            'start_date': self._quarter_start(segment[0]),
            'end_date': segment[-1]
        })
    return queries

def _are_adjacent_quarters(self, period1: str, period2: str) -> bool:
    """判断两个报告期是否相邻"""
    year1, q1 = int(period1[:4]), int(period1[4:6]) // 3
    year2, q2 = int(period2[:4]), int(period2[4:6]) // 3
    return (year1 * 4 + q1 + 1) == (year2 * 4 + q2)
```
