# 增量下载去重功能问题修复方案

**文档日期**: 2026-02-15  
**问题来源**: 增量下载去重功能测试报告  
**文档状态**: 待修复

---

## 一、问题总结

根据测试报告，增量下载去重功能存在三个核心问题：

1. **问题1**: `get_stock_existing_dates()` 无法正确读取已有数据（影响 Type A/B/C 所有接口）
2. **问题2**: Type C 接口日期范围传递不完整（影响 stk_rewards 等接口）
3. **问题3**: 精确查询无法避免 API 返回超范围数据（影响 Type B 接口）

---

## 二、问题详细分析与修复方案

### 问题1: get_stock_existing_dates 缓存机制问题

#### 2.1.1 问题描述

**现象**: 
- 第一次下载后，数据已保存到 parquet 文件
- 第二次下载时，缺口检测仍然报告"股票无数据"或"缺失 N 个日期"
- 日志显示 `get_stock_existing_dates()` 返回空集合

**影响接口**: 所有 Type A/B/C 接口（cyq_chips, income_vip, cashflow_vip, top10_holders 等）

#### 2.1.2 根本原因分析

**位置**: `app4/core/coverage_manager.py:640-692`

```python
def get_stock_existing_dates(self, interface_name, ts_code, date_column='trade_date'):
    cache_key = f"{interface_name}:{ts_code}:dates"
    
    with self._cache_lock:
        if cache_key in self._cache:
            return self._cache[cache_key]  # <-- 问题：缓存可能存储了空集合
```

**潜在问题**:

1. **缓存污染**: 如果第一次读取时 parquet 文件尚未完全写入，可能缓存空集合
2. **进程间缓存不共享**: 每次运行是独立进程，缓存应该为空，但可能有持久化机制
3. **文件读取时序**: `read_interface_data` 可能读取到旧版本文件
4. **文件名过滤问题**: `read_interface_data` 中的文件名过滤可能错误地排除了有效文件

**关键疑点**: 从日志看，第二次运行时 `get_stock_existing_dates()` 返回空，但实际数据文件存在。

#### 2.1.4 修复方案

**方案 A: 强制绕过缓存（临时调试）**

```python
def get_stock_existing_dates(
    self,
    interface_name: str,
    ts_code: str,
    date_column: str = 'trade_date',
    skip_cache: bool = False  # 新增参数
) -> Set[str]:
    """
    获取指定股票已存在的所有日期
    
    Args:
        skip_cache: 是否跳过缓存（用于调试）
    """
    cache_key = f"{interface_name}:{ts_code}:dates"
    
    # [调试] 添加详细日志
    logger.debug(f"[{interface_name}/{ts_code}] 获取已有日期 - 缓存键: {cache_key}")
    
    with self._cache_lock:
        if not skip_cache and cache_key in self._cache:
            cached_dates = self._cache[cache_key]
            logger.debug(f"  缓存命中: {len(cached_dates)} 个日期")
            return cached_dates
        logger.debug(f"  缓存未命中，从存储读取")

    try:
        # [修复] 先检查目录是否存在
        import os
        data_dir = os.path.join(self.storage_manager.storage_dir, interface_name)
        if not os.path.exists(data_dir):
            logger.debug(f"  数据目录不存在: {data_dir}")
            return set()
            
        # [修复] 列出目录中的所有文件
        files = os.listdir(data_dir)
        parquet_files = [f for f in files if f.endswith('.parquet')]
        logger.debug(f"  找到 {len(parquet_files)} 个 parquet 文件: {parquet_files}")

        # 读取数据
        df = self.storage_manager.read_interface_data(
            interface_name,
            columns=[date_column, 'ts_code']
        )
        
        logger.debug(f"  读取到 {len(df)} 行原始数据")
        
        if df.is_empty():
            logger.debug(f"  数据为空，返回空集合")
            return set()

        # 过滤特定股票
        import polars as pl
        filtered = df.filter(pl.col('ts_code') == ts_code)
        
        logger.debug(f"  过滤 ts_code={ts_code} 后: {len(filtered)} 行")

        if filtered.is_empty():
            logger.debug(f"  过滤后为空，可能 ts_code 不匹配")
            # [调试] 显示所有可用的 ts_code
            all_ts_codes = df['ts_code'].unique().to_list()
            logger.debug(f"  可用的 ts_code: {all_ts_codes}")
            return set()

        # 格式化日期
        dates = set()
        for date_val in filtered[date_column]:
            formatted = format_date(date_val)
            if formatted:
                dates.add(formatted)

        logger.info(f"[{interface_name}/{ts_code}] 读取到 {len(dates)} 个已有日期: {sorted(dates)}")

        # 写入缓存
        with self._cache_lock:
            self._cache[cache_key] = dates

        return dates

    except Exception as e:
        logger.error(f"获取 {interface_name}/{ts_code} 的现有日期失败: {e}")
        logger.exception(e)  # 打印完整堆栈
        return set()
```

### 问题3: 精确查询无法避免 API 返回超范围数据

#### 2.3.1 问题描述

**现象**: 
- 使用精确查询 `20230101 ~ 20230331` 期望只获取 20230331 报告期数据
- 但 TuShare API 返回了 `20221231`（前一季度）的数据
- 导致存储层去重时发现 1 条重复数据

**影响接口**: Type B 接口（income_vip, cashflow_vip 等）

#### 2.3.2 根本原因分析

**位置**: `app4/core/coverage_manager.py:931-958`

```python
# 精确查询策略
for period in missing_periods:
    # period 格式：YYYYMMDD
    period_year = period[:4]
    period_month_day = period[4:]
    
    # 计算该季度的开始日期
    quarter_start_map = {
        '0331': f'{period_year}0101',
        '0630': f'{period_year}0401',
        '0930': f'{period_year}0701',
        '1231': f'{period_year}1001',
    }
    period_start = quarter_start_map.get(period_month_day, f'{period_year}0101')
    
    precise_tasks.append({
        'ts_code': ts_code,
        'start_date': period_start,
        'end_date': period
    })
```

**问题**: 
- TuShare API 的 `start_date` 和 `end_date` 是过滤条件，不是精确匹配
- API 可能返回在此范围内的所有报告期数据
- 例如查询 `20230101 ~ 20230331` 可能返回 `20221231` 的数据

#### 2.3.3 修复方案

**方案 A: 下载后过滤（推荐）**

在下载数据后，过滤掉不在期望报告期列表中的记录：

```python
# app4/core/downloader.py 或 coverage_manager.py

def _filter_unexpected_periods(df, expected_periods, period_column='end_date'):
    """
    过滤掉不在期望报告期列表中的数据
    
    Args:
        df: Polars DataFrame
        expected_periods: 期望的报告期列表 ['20230331', '20230630', ...]
        period_column: 报告期列名
    
    Returns:
        过滤后的 DataFrame
    """
    if df.is_empty() or not expected_periods:
        return df
    
    import polars as pl
    
    # 转换为集合提高查找效率
    expected_set = set(expected_periods)
    
    # 过滤数据
    original_count = len(df)
    filtered = df.filter(pl.col(period_column).is_in(list(expected_set)))
    filtered_count = len(filtered)
    
    if original_count != filtered_count:
        removed_periods = set(df[period_column].unique().to_list()) - expected_set
        logger.info(f"过滤掉 {original_count - filtered_count} 条超范围数据，"
                   f"不期望的报告期: {removed_periods}")
    
    return filtered
```

**方案 C: 在存储层增强去重**

确保存储层的去重功能能正确处理这种情况：

```python
# app4/core/dedup.py 或 storage.py

def deduplicate_by_primary_key(df, interface_config):
    """
    根据主键去重，保留最新数据
    """
    primary_keys = interface_config.get('output', {}).get('primary_key', [])
    
    if not primary_keys or df.is_empty():
        return df
    
    # 只保留存在于 DataFrame 中的主键列
    existing_keys = [k for k in primary_keys if k in df.columns]
    
    if not existing_keys:
        return df
    
    # 按 _update_time 排序，保留最新
    if '_update_time' in df.columns:
        df = df.sort('_update_time', descending=False)
    
    # 去重
    original_count = len(df)
    df = df.unique(subset=existing_keys, keep='last')
    
    logger.info(f"去重: 原始 {original_count} 条 -> 最终 {len(df)} 条")
    
    return df
```
