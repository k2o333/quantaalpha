# 增量下载去重功能问题根本原因诊断

## 问题概述
测试报告显示，第二次下载时系统未能正确检测到已有数据，导致全量重新下载，去重发生在存储层而非下载层。

---

## 主要问题：路径解析与工作目录不一致

### 1. 配置使用相对路径
**文件**: `app4/config/settings.yaml:3`  
```yaml
storage:
  base_dir: "data"  # ❌ 相对路径，依赖于工作目录
  format: "parquet"
  batch_size: 10000
```

### 2. StorageManager 初始化
**文件**: `app4/main.py:214`  
```python
storage_manager = StorageManager(
    processor=processor,
    config_loader=config_loader,
    storage_dir=config_loader.global_config.get('storage', {}).get('base_dir', '../data'),
    # ...
)
```

### 3. 问题发生流程

#### 场景 1：从 app4 目录运行
```bash
cd /home/quan/testdata/aspipe_v4/app4
python main.py
```
- 相对路径 `"data"` 解析为：`/home/quan/testdata/aspipe_v4/app4/data`
- 数据成功保存到该目录

#### 场景 2：从项目根目录运行
```bash
cd /home/quan/testdata/aspipe_v4
python app4/main.py
```
- 相对路径 `"data"` 解析为：`/home/quan/testdata/aspipe_v4/data`
- 该目录为空或不存在
- **`get_stock_existing_dates()` 读不到已有数据，返回空集合**
- 缺口检测认为需要全量下载

### 4. 证据
项目中存在两个 data 目录：
- `/home/quan/testdata/aspipe_v4/app4/data` - 实际保存数据的位置
- `/home/quan/testdata/aspipe_v4/data` - 根目录下可能残留或误创建的位置

## 次要问题：缓存清除不完整

### 问题描述
`clear_dates_cache()` 方法只清除了一个缓存字典，没有清除另一个。

**文件**: `app4/core/coverage_manager.py:597-610`  
```python
def clear_dates_cache(self, interface_name: str = None):
    """
    清除日期缓存（在数据写入后调用）
    """
    with self._cache_lock:
        if interface_name:
            # 只清除指定接口的缓存
            self._existing_dates_cache.pop(interface_name, None)
            # ❌ 缺少清除 _cache 中的 {interface_name}:{ts_code}:dates 缓存
        else:
            # 清除所有缓存
            self._existing_dates_cache.clear()
            # ❌ 缺少清除 _cache 中的所有缓存
```

### 受影响的缓存键
`get_stock_existing_dates()` 使用的缓存键格式：
```python
cache_key = f"{interface_name}:{ts_code}:dates"
```

## 其他小问题

### 空结果未缓存
**文件**: `app4/core/coverage_manager.py:669 和 676`  
当 `df.is_empty()` 或 `filtered.is_empty()` 时，直接返回空集合但没有存入缓存。这对功能影响不大，但不够一致。

---

## 修复建议

### 方案 1：使用绝对路径（推荐）
修改 `app4/config/settings.yaml` 或 `app4/main.py`，使用基于 `__file__` 的绝对路径：
```python
# 在 main.py 中
import os
app_dir = os.path.dirname(os.path.abspath(__file__))
storage_dir = os.path.join(app_dir, 'data')  # 绝对路径
```

### 方案 2：修复缓存清除
修改 `clear_dates_cache()` 方法，同时清除两个缓存字典。

### 方案 3：统一运行入口
确保始终从 `app4` 目录运行程序，或修改启动脚本固定工作目录。
