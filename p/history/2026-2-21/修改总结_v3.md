# App4 日期锚定接口参数传递修复总结报告（v3）

## 修改概述

根据 `/home/quan/testdata/aspipe_v4/p/2026-2-21/修改方案_最终版_v3.md` 文件，成功实施了对 `cyq_perf` 接口的完整修复。v3版本特别解决了配置迁移过程中 `mode` 字段丢失的关键问题。

## 根本原因分析

### 问题根源
在配置迁移过程中，`migrate_legacy_config` 函数在处理 `reverse_date_range` 模式时，没有保留 `mode` 字段，导致 `PaginationComposer.compose` 方法无法正确识别该模式。

### 问题流程
1. `cyq_perf` 接口配置：`{mode: "reverse_date_range", ...}`
2. `migrate_legacy_config` 转换后：`{time_range: {...}}`（**丢失了 `mode` 字段！**）
3. `compose()` 方法中 `self.config.get("mode", "")` 返回 `""` 而非 `"reverse_date_range"`
4. 条件检查 `pagination_mode == "reverse_date_range"` 失败
5. 跳过 `_apply_date_anchor_range` 处理
6. API 仍然收到错误格式的参数

### 3. **v3 关键修改：修复配置迁移函数**

#### 3.1 修改 migrate_legacy_config 函数
**修改前（第 689-695 行）：**
```python
elif mode == "reverse_date_range":
    new_config["time_range"] = {
        "enabled": True,
        "window": window_str,
        "reverse": True,
        "stop_on_empty": old_pagination.get("empty_threshold_days", 90),
    }
    # 问题：没有保留 mode 字段！
```

**修改后：**
```python
elif mode == "reverse_date_range":
    new_config["mode"] = "reverse_date_range"  # 新增：保留 mode 字段，用于 compose() 判断
    new_config["time_range"] = {
        "enabled": True,
        "window": window_str,
        "reverse": True,
        "stop_on_empty": old_pagination.get("empty_threshold_days", 90),
    }
```

**关键对比**：这与 `period_range` 模式（第 711-715 行）的处理方式保持一致，都保留了 `mode` 字段。
