## 四、分段逻辑优化

### 1. 问题分析

当前 `_merge_dates_to_ranges` 方法使用"间隔 > 3 天"的规则分段，逻辑不合理：

```python
# 当前代码 (coverage_manager.py:1255)
if (curr_dt - prev_dt).days <= 3:
    range_end = curr
else:
    ranges.append((range_start, range_end))
```

**问题示例**：

```
时间轴:  01  02  03  04  05  06  07  08  09  10
已有数据:                  ██  ██                  (05, 06 有数据)

【当前逻辑】(间隔>3天分段):
缺失: 01,02,03,04,07,08,09,10
间隔都≤3天 → 合并为1段: 01~10 (错误：会把已有数据覆盖)

【期望逻辑】(按空洞分段):
空洞1: 01~04  → 1段
空洞2: 07~10  → 1段
结果: 2段
```

### 2. 期望行为

根据已有数据边界切分空洞，确保每个下载段不覆盖已有数据。

### 3. 新算法设计

```
输入：
- trade_days: [20260101, 20260102, 20260103, 20260106, 20260107, ...]
- existing_dates: {20260105, 20260106}

输出：
- 空洞区间: [(20260101, 20260104), (20260107, ...)]

算法：
遍历 trade_days:
  if 当前日期不在 existing_dates 中:
    if 正在追踪区间: 
      扩展区间结束日期
    else:
      开始新区间
  else:  # 当前日期有数据
    if 正在追踪区间:
      输出区间，结束追踪
```

### 4. 示意图对比

```
时间轴:  20260101 ─── 20260104   20260105   20260106   20260107 ─── 20260110
                      ↑ 空洞1 ↑    ██已有██   ██已有██    ↑ 空洞2 ↑

【期望分段结果】:
- 第1段: 20260101 ~ 20260104
- 第2段: 20260107 ~ 20260110
```

### 5. 推荐代码修改

修改 `app4/core/coverage_manager.py` 中的 `_merge_dates_to_ranges` 方法：

```python
def _merge_dates_to_ranges(
    self, 
    dates: List[str], 
    existing_dates: Optional[Set[str]] = None
) -> List[tuple]:
    """
    将缺失日期按空洞边界合并为连续区间
    
    Args:
        dates: 缺失的日期列表（需已排序）
        existing_dates: 已有数据日期集合，用于确定空洞边界
    
    Returns:
        连续区间列表，每个元素为 (start_date, end_date)
    
    注意：
        如果传入 existing_dates，则按空洞边界分段
        如果不传入，则退化为原有逻辑（间隔>3天分段）
    """
    if not dates:
        return []
    
    sorted_dates = sorted(dates)
    
    # 兼容原有调用方式：不传入 existing_dates 时使用旧逻辑
    if existing_dates is None:
        return self._merge_by_interval(sorted_dates)
    
    # 新逻辑：按空洞边界分段
    ranges = []
    range_start = None
    range_end = None
    
    for date in sorted_dates:
        if date not in existing_dates:
            # 当前日期缺失
            if range_start is None:
                range_start = date
                range_end = date
            else:
                range_end = date
        else:
            # 当前日期有数据，结束当前区间
            if range_start is not None:
                ranges.append((range_start, range_end))
                range_start = None
                range_end = None
    
    # 处理最后一个区间
    if range_start is not None:
        ranges.append((range_start, range_end))
    
    return ranges

def _merge_by_interval(self, sorted_dates: List[str]) -> List[tuple]:
    """原有逻辑：按间隔>3天分段（保留向后兼容）"""
    ranges = []
    range_start = sorted_dates[0]
    range_end = sorted_dates[0]
    
    for i in range(1, len(sorted_dates)):
        curr = sorted_dates[i]
        prev = sorted_dates[i - 1]
        
        curr_dt = datetime.strptime(curr, "%Y%m%d")
        prev_dt = datetime.strptime(prev, "%Y%m%d")
        
        if (curr_dt - prev_dt).days <= 3:
            range_end = curr
        else:
            ranges.append((range_start, range_end))
            range_start = curr
            range_end = curr
    
    ranges.append((range_start, range_end))
    return ranges
```

同时修改 `_detect_trade_date_gaps` 方法中的调用：

## 五、潜在风险

### 1. 性能考虑

对于上市时间较早的股票（如 1991 年上市的平安银行），全量缺口检测会扫描 35 年的交易日历数据。

### 2. 影响范围

修改后影响所有 Type A 接口的 `--update` 模式行为：

| 接口 | 数据日期字段 |
|------|-------------|
| `cyq_chips` | `trade_date` |
| `moneyflow_dc` | `trade_date` |
| `stk_factor_pro` | `trade_date` |
