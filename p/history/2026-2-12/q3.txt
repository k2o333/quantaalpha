### 2. 具体代码证据

#### A. 故障点：`main.py` 和 `params_builder.py`
程序员指出 `run_update_mode` 直接使用了 `ParamsBuilder`，导致缺口检测失效。

*   **[main.py](file:///home/quan/testdata/aspipe_v4/app4/main.py)**:
    在 `run_update_mode` 中，代码逻辑如下：
    ```python
    if result.requires_stock_loop:
        # ... 获取股票列表 ...
        # 直接构建参数列表，完全没有调用 detect_stock_gaps
        params_list = builder.build_params_list(result, stock_list)
        # 直接开始下载
        downloaded_count = run_concurrent_stock_download(...)
    ```

*   **[params_builder.py](file:///home/quan/testdata/aspipe_v4/app4/core/params_builder.py)**:
    `build_params_list` 方法（及其调用的 `_build_stock_loop_date_params_list` 等）仅仅是简单的遍历股票列表并复制参数，**没有任何与数据库交互或检查现有数据的逻辑**。
    ```python
    def _build_stock_loop_date_params_list(self, result, stock_list):
        params_list = []
        for stock in stock_list:
            p = result.params.copy()
            # 仅设置默认起始日期，不检查已有数据
            if 'start_date' not in p:
                p['start_date'] = stock.get('list_date', '20050101')
            params_list.append(p)
        return params_list
    ```

#### B. 现有能力：`coverage_manager.py` 和 `pagination.py`
程序员指出缺口检测逻辑是存在的，只是未被调用。
