# 修正方案：参数语义感知的接口参数处理

## 问题诊断

### 原始问题
6个接口存在参数定义与代码强制传递不匹配的问题：

| 接口 | 配置定义 | 代码强制传递 | 结果 |
|------|---------|-------------|------|
| **disclosure_date** | 无 `start_date`，`end_date`=财报周期 | `start_date` + `end_date` | 查询未来日期，返回0条 |
| **dividend** | 无 `start_date`，无 `end_date` | `start_date` + `end_date` | 参数被忽略，返回0条 |
| **pledge_detail** | 只有 `ts_code` | `start_date` + `end_date` | 参数被忽略 |
| **pledge_stat** | 无 `start_date`，`end_date`=截止日期 | `start_date` + `end_date` | 查询未来日期，返回0条 |
| **stk_rewards** | 无 `start_date`，`end_date`=报告期 | `start_date` + `end_date` | 查询未来日期，返回0条 |
| **top10_holders** | 无 `start_date`，使用 `period` | `start_date` + `end_date` | 使用默认period，参数被忽略 |

### 混合方案的问题

混合方案尝试根据接口配置动态设置 `end_date`：

```python
if 'end_date' in parameter_config and 'end_date' not in stock_params:
    stock_params['end_date'] = datetime.now().strftime('%Y%m%d')
```

**但这会导致新的问题**：

对于 `disclosure_date`，`end_date` 的语义是**财报周期**（如 20241231 表示2024年年报），而不是查询结束日期。传递当前日期（如 20260201）会导致 API 返回空结果。

### 根本原因

1. **参数语义混淆**：`end_date` 在不同接口中有不同含义
   - `daily` 接口：`end_date` = 查询结束日期
   - `disclosure_date` 接口：`end_date` = 财报周期（必须是季度末日期）
   - `pledge_stat` 接口：`end_date` = 截止日期（特定日期）

2. **自动填充策略不当**：代码不应该为所有支持 `end_date` 的接口自动填充当前日期

## 修正方案

### 设计原则

1. **不自动填充语义特殊的参数**：对于语义非"查询范围"的参数，不自动填充
2. **仅传递用户显式指定的参数**：除非参数语义明确为日期范围，否则不自动添加
3. **配置驱动 + 语义感知**：根据参数描述判断是否可以自动填充

### 参数分类

| 参数类型 | 示例 | 自动填充策略 |
|---------|------|-------------|
| 查询日期范围 | `start_date` + `end_date` (daily接口) | 可以自动填充 |
| 财报周期 | `end_date` (disclosure_date接口) | **不自动填充** |
| 报告期 | `end_date` (stk_rewards接口) | **不自动填充** |
| 截止日期 | `end_date` (pledge_stat接口) | **不自动填充** |
| 无日期参数 | dividend接口 | 不传递日期参数 |

### 实现代码

#### 修改位置1：app4/main.py

**位置**：第608-615行（参数构建逻辑）

```python
# 检查分页模式
pagination_config = interface_config.get('pagination', {})
is_stock_loop = (
    pagination_config.get('enabled', False) and
    pagination_config.get('mode') == 'stock_loop'
)

if is_stock_loop:
    # stock_loop 模式：不传递日期参数，让接口返回全历史
    params = {}
    if args.ts_code:
        params['ts_code'] = args.ts_code
    logger.info(f"Using stock_loop mode for {interface_name}, fetching full history")
else:
    # 其他模式：传递标准日期参数
    params = {
        'start_date': args.start_date,
        'end_date': args.end_date
    }
    if args.ts_code:
        params['ts_code'] = args.ts_code
```

## 特殊情况处理

### 如果用户需要查询特定财报周期

用户可以通过显式参数传递：

```bash
# 查询2024年年报的披露计划
python app4/main.py --interface disclosure_date --ts_code 000001.SZ --end_date 20241231
```

需要在 main.py 中添加对额外参数的支持：

```python
# 添加对其他参数的支持
for param_name in parameter_config:
    if param_name not in ['ts_code', 'start_date', 'end_date']:
        param_value = getattr(args, param_name, None)
        if param_value:
            params[param_name] = param_value
```

## 风险控制

### 潜在风险

1. **数据量增加**：不再按日期过滤，可能下载更多历史数据
2. **API 调用次数**：单只股票可能需要多次调用（分页）

### 缓解措施

1. **覆盖率管理**：利用 `coverage_manager` 跳过已存在的股票
2. **增量模式**：支持 `--incremental` 只下载缺失数据
3. **用户控制**：用户仍可通过 `--ts_code` 指定单只股票
