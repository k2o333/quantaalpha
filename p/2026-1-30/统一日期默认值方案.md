# 统一日期默认值方案

## 方案概述

将 `tscode_historical` 组接口的特殊日期处理逻辑移除，统一使用 `19900101` 作为所有接口的默认起始日期。

## 背景

当前代码存在两套日期默认值逻辑：
1. **一般接口**：默认从 `20230101` 开始
2. **`tscode_historical` 组接口**：默认从 `19900101` 开始（全历史）

这种差异化处理增加了代码复杂度，且 `20230101` 作为默认值对大多数历史数据接口而言过于保守。

## 修改内容

### 1. 修改 `app4/main.py`

#### 修改点 A：调整命令行参数默认值
**位置**：第171行

**修改前**:
```python
parser.add_argument('--start_date', type=str, default='20230101',
                    help='起始日期 (YYYYMMDD)')
```

**修改后**:
```python
parser.add_argument('--start_date', type=str, default='19900101',
                    help='起始日期 (YYYYMMDD)')
```

#### 修改点 B：移除 `tscode_historical` 组特殊处理
**位置**：第591-600行

**修改前**:
```python
# [新增] 检查是否是 tscode_historical 接口
tscode_historical_group = config_loader.global_config.get('groups', {}).get('tscode_historical', [])
is_tscode_historical_interface = interface_name in tscode_historical_group

# [新增] 对于 tscode_historical 接口，如果用户使用默认日期，则改为 1990年1月1日
if is_tscode_historical_interface:
    if args.start_date == '20230101' and args.end_date is None:
        logger.info(f"Using default date range for tscode_historical interface {interface_name}: 19900101 to today")
        args.start_date = '19900101'
        args.end_date = datetime.now().strftime('%Y%m%d')
```

**修改后**:
```python
# 日期参数已通过命令行默认值统一设置为 19900101
# 移除 tscode_historical 组的特殊处理逻辑
```

或直接删除该代码块。

## 影响分析

### 受影响接口

所有原本在 `tscode_historical` 组中的接口：
- `stk_rewards`, `income_vip`, `balancesheet_vip`, `cashflow_vip`
- `fina_indicator_vip`, `fina_mainbz_vip`, `forecast_vip`, `express_vip`
- `fina_audit`, `disclosure_date`, `top10_floatholders`, `top10_holders`
- `pledge_stat`, `pledge_detail`, `dividend`, `stk_factor`, `stk_factor_pro`

### 行为变化

| 场景 | 修改前 | 修改后 |
|------|--------|--------|
| 不指定 `--start_date` | 一般接口从2023年开始，tscode_historical组从1990年开始 | 所有接口统一从1990年开始 |
| 指定 `--start_date 20200101` | 从2020年开始 | 无变化，仍从2020年开始 |
| 指定 `--ts_code` 但不指定日期 | 根据接口类型决定起始日期 | 统一从1990年开始查该股票历史 |

### 数据下载量变化

- **增加**：原本使用默认值的非 tscode_historical 接口（如 `daily`, `daily_basic` 等）会下载更多历史数据
- **不变**：原本就在 tscode_historical 组中的接口行为保持一致

## 实施步骤

1. **修改 `app4/main.py`**
   - 将第171行的 `default='20230101'` 改为 `default='19900101'`
   - 删除或注释掉第591-600行的 `tscode_historical` 特殊处理代码

2. **测试验证**
   ```bash
   # 测试一般接口（以前从2023开始，现在应从1990开始）
   python app4/main.py --interface daily --ts_code 000001.SZ --log-level DEBUG
   
   # 测试 tscode_historical 组接口（行为应保持不变）
   python app4/main.py --interface fina_audit --ts_code 000001.SZ --log-level DEBUG
   
   # 测试显式指定日期（应正常工作）
   python app4/main.py --interface daily --ts_code 000001.SZ --start_date 20230101 --end_date 20231231
   ```

3. **可选配置清理**
   - 如确认 `tscode_historical` 组定义不再被其他代码引用，可从 `settings.yaml` 中移除

## 风险控制

### 潜在风险
1. **数据下载量增加**：默认下载更多历史数据可能导致：
   - API 调用次数增加
   - 下载时间延长
   - 存储空间占用增加

2. **API 积分消耗**：TuShare 积分有限的用户可能更快耗尽每日配额

### 缓解措施
1. **用户仍可通过参数控制范围**：
   ```bash
   # 只下载近期数据
   python app4/main.py --interface daily --start_date 20240101
   ```

2. **建议搭配覆盖率管理**：
   - 使用 `--incremental` 模式避免重复下载
   - 利用 coverage_manager 跳过已存在的数据
