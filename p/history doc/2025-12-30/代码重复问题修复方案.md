## 问题 1：日志配置重复

### 当前状态

**settings.yaml** (已存在完整配置)：
```yaml
logging:
  level: "INFO"
  file: "../log/aspipe_v4.log"
  max_size_mb: 100
  backup_count: 5
```

**main.py** (硬编码)：
```python
def setup_logging(log_level: str = "INFO", log_file: str = "../log/aspipe_v4.log"):
    # ...

# 调用时
log_config = {
    'log_level': args.log_level,
    'log_file': '../log/aspipe_v4.log'  # 硬编码，未从 settings.yaml 读取
}
setup_logging(**log_config)
```

### 修复方案

**修改 main.py 的 setup_logging 函数**：

```python
def setup_logging(log_config: dict):
    """设置日志配置

    Args:
        log_config: 日志配置字典，包含 level, file, max_size_mb, backup_count
    """
    log_level = log_config.get('level', 'INFO')
    log_file = log_config.get('file', '../log/aspipe_v4.log')
    max_size_mb = log_config.get('max_size_mb', 100)
    backup_count = log_config.get('backup_count', 5)

    # 确保日志目录存在
    log_dir = os.path.dirname(log_file)
    os.makedirs(log_dir, exist_ok=True)

    # 创建日志格式器
    formatter = logging.Formatter(
        '%(asctime)s - %(name)s - %(levelname)s - %(message)s'
    )

    # 创建文件处理器（支持轮转）
    from logging.handlers import RotatingFileHandler
    file_handler = RotatingFileHandler(
        log_file,
        maxBytes=max_size_mb * 1024 * 1024,
        backupCount=backup_count,
        encoding='utf-8'
    )
    file_handler.setLevel(getattr(logging, log_level))
    file_handler.setFormatter(formatter)

    # 创建控制台处理器
    console_handler = logging.StreamHandler()
    console_handler.setLevel(getattr(logging, log_level))
    console_handler.setFormatter(formatter)

    # 配置根日志记录器
    root_logger = logging.getLogger()
    root_logger.setLevel(getattr(logging, log_level))
    root_logger.addHandler(file_handler)
    root_logger.addHandler(console_handler)
```

**修改 main 函数中的调用**：

```python
# 设置日志 - 从 settings.yaml 读取配置
logging_config = config_loader.global_config.get('logging', {})
setup_logging(logging_config)
```

### 修复效果

- 日志配置统一在 settings.yaml 管理
- 支持日志文件轮转（max_size_mb, backup_count）
- 命令行参数 --log-level 可以覆盖配置文件中的 level

## 问题 2：数据处理和保存逻辑重复

### 当前状态

**并发模式** (main.py:293-298)：
```python
df = processor.process_data(all_data, interface_config)
validation_result = processor.validate_data(df, interface_config)
storage_manager.save_data(interface_name, df.to_dict('records'), async_write=True)
logger.info(f"Saved {len(df)} processed records for {interface_name}")
if validation_result['duplicate_records'] > 0:
    logger.info(f"Found {validation_result['duplicate_records']} duplicate records for {interface_name}")
```

**普通模式** (main.py:320-325)：完全相同的代码

### 修复方案

**在 main.py 中添加独立函数**：

```python
def process_and_save_data(data, interface_name, interface_config, processor, storage_manager):
    """处理并保存数据的通用函数

    Args:
        data: 原始数据列表
        interface_name: 接口名称
        interface_config: 接口配置
        processor: 数据处理器
        storage_manager: 存储管理器

    Returns:
        处理后的 DataFrame，如果处理失败则返回 None
    """
    if not data:
        logger.warning(f"No data to process for {interface_name}")
        return None

    # 处理数据
    df = processor.process_data(data, interface_config)
    validation_result = processor.validate_data(df, interface_config)

    # 保存数据
    storage_manager.save_data(interface_name, df.to_dict('records'), async_write=True)

    logger.info(f"Saved {len(df)} processed records for {interface_name}")
    if validation_result['duplicate_records'] > 0:
        logger.info(f"Found {validation_result['duplicate_records']} duplicate records for {interface_name}")

    return df
```

**替换并发模式中的代码**：

```python
if all_data:
    logger.info(f"Successfully downloaded {len(all_data)} total records for {interface_name}")
    process_and_save_data(all_data, interface_name, interface_config, processor, storage_manager)
else:
    logger.warning(f"No data downloaded for {interface_name}")
```

**替换普通模式中的代码**：

```python
if data:
    logger.info(f"Successfully downloaded {len(data)} records for {interface_name}")
    process_and_save_data(data, interface_name, interface_config, processor, storage_manager)
else:
    logger.warning(f"No data downloaded for {interface_name}")
```

### 修复效果

- 消除代码重复，提高可维护性
- 数据处理逻辑统一管理
- 便于后续扩展（如添加统一的错误处理、监控等）

## 问题 3：Token 环境变量名不一致

### 当前状态

- **settings.yaml**: `${TUSHARE_TOKEN}` (大写)
- **downloader.py:361**: `os.getenv('tushare_token', '')` (小写)
- **config_loader.py:61-62**: 同时检查两种写法

### 修复方案

**统一使用大写 TUSHARE_TOKEN**

**修改 downloader.py:361**：

```python
# 原代码：
token = os.getenv('tushare_token', '')

# 修改为：
token = os.getenv('TUSHARE_TOKEN', '')
```

**修改 config_loader.py:61-62**：

```python
# 原代码：
logger.debug("  os.getenv('tushare_token'): %s", os.getenv('tushare_token', 'NOT FOUND')[:10] if os.getenv('tushare_token') else 'NOT FOUND')
logger.debug("  os.getenv('TUSHARE_TOKEN'): %s", os.getenv('TUSHARE_TOKEN', 'NOT FOUND')[:10] if os.getenv('TUSHARE_TOKEN') else 'NOT FOUND')

# 修改为：
token = os.getenv('TUSHARE_TOKEN', '')
logger.debug("  TUSHARE_TOKEN: %s", token[:10] if token else 'NOT FOUND')
```

**更新 .env 文件**（如果存在）：

```bash
# 统一使用大写
TUSHARE_TOKEN=your_token_here
```

### 修复效果

- 环境变量命名统一，避免混淆
- 符合环境变量通常使用大写的惯例
- 代码更清晰易读

## 补充问题：RateLimiter 初始化重复

### 当前状态

**全局限流器** (main.py:131)：
```python
global_rate_limit = config_loader.global_config.get('request', {}).get('rate_limit', 60)
global_rate_limiter = RateLimiter(global_rate_limit)
```

**接口级限流器** (main.py:307-308)：
```python
interface_rate_limit = interface_config.get('permissions', {}).get('rate_limit', global_rate_limit)
interface_rate_limiter = RateLimiter(interface_rate_limit)
```

### 修复方案（可选）

由于只有2处重复，且代码逻辑简单，**不建议过度封装**。如果确实需要统一，可以添加辅助函数：

```python
def create_rate_limiter(rate_limit: int, name: str = "default") -> RateLimiter:
    """创建并配置限流器"""
    logger.debug(f"Creating rate limiter '{name}' with limit: {rate_limit}")
    return RateLimiter(rate_limit)
```

但考虑到代码重复度低（仅2处）且逻辑简单，**保持现状即可**。

### 修复效果

- 如需封装可提高一致性
- 但会增加不必要的抽象层
