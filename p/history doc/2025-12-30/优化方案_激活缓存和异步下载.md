# aspipe_v4 优化方案：激活缓存和异步下载（最小化修改版）

## 概述

当前 aspipe_v4 App4 存在以下问题：
1. **数据缓存未激活**：缓存功能已实现但被人为禁用（调试注释）
2. **下载同步串行**：虽然有 ThreadPoolExecutor，但在 main.py 中未使用
3. **资源浪费**：重复运行时重新下载数据，浪费时间

### 2. 下载现状

#### 已实现的并发能力

**位置**：`app4/core/scheduler.py`

```python
class TaskScheduler:
    def __init__(self, max_workers: int = 8):
        self.executor = ThreadPoolExecutor(max_workers=max_workers)

    def submit_tasks(self, tasks: List[Dict[str, Any]]) -> List[Any]:
        """批量提交任务到线程池"""
        futures = []
        for task in tasks:
            func = task['func']
            args = task.get('args', ())
            kwargs = task.get('kwargs', {})
            future = self.executor.submit(func, *args, **kwargs)
            futures.append(future)

        results = []
        for future in as_completed(futures):
            try:
                result = future.result()
                results.append(result)
            except Exception as e:
                logger.error(f"Task execution error: {str(e)}")
                results.append(None)
        return results
```

```python
def download_single_stock(self, interface_config: Dict[str, Any], params: Dict[str, Any]) -> List[Dict[str, Any]]:
    """
    下载单只股票的数据（用于并发下载）

    Args:
        interface_config: 接口配置
        params: 请求参数（必须包含 ts_code）

    Returns:
        返回的数据
    """
    api_name = interface_config.get('api_name')
    ts_code = params.get('ts_code')

    logger.info(f"Downloading {api_name} for {ts_code}")

    # 执行日期范围分页下载
    data = self._execute_date_range_pagination(interface_config, params)

    if data:
        logger.debug(f"Downloaded {len(data)} records for {ts_code}")

    return data
```

##### 2.3 添加并发下载辅助函数

**文件**：`app4/main.py`

**新增函数**（在 `main()` 函数之前添加）：
```python
def _download_stocks_concurrently(
    downloader, interface_config, interface_name, params,
    scheduler, rate_limiter, processor, storage_manager
) -> List[Dict[str, Any]]:
    """
    并发下载多只股票的数据

    Args:
        downloader: 下载器实例
        interface_config: 接口配置
        interface_name: 接口名称
        params: 基础请求参数
        scheduler: 任务调度器
        rate_limiter: 速率限制器
        processor: 数据处理器
        storage_manager: 存储管理器

    Returns:
        所有股票的数据
    """
    logger.info(f"Starting concurrent download for {interface_name}...")

    stock_list = downloader.cache_manager.get_stock_list()
    if not stock_list:
        logger.info("Cache miss for stock list, fetching from API...")
        stock_list = downloader._make_request(
            downloader.config_loader.get_interface_config('stock_basic'),
            {'list_status': 'L'}
        )
        if stock_list:
            downloader.cache_manager.set_stock_list(stock_list)

    if not stock_list:
        logger.error("Failed to get stock list")
        return []

    max_workers = scheduler.max_workers
    batch_size = max_workers
    total_batches = (len(stock_list) + batch_size - 1) // batch_size

    logger.info(f"Processing {len(stock_list)} stocks in {total_batches} batches (batch_size={batch_size})")

    all_data = []

    for batch_idx in range(total_batches):
        start_idx = batch_idx * batch_size
        end_idx = min(start_idx + batch_size, len(stock_list))
        batch_stocks = stock_list[start_idx:end_idx]

        logger.info(f"Processing batch {batch_idx + 1}/{total_batches} (stocks {start_idx + 1}-{end_idx})")

        tasks = []
        for stock in batch_stocks:
            stock_params = params.copy()
            stock_params['ts_code'] = stock['ts_code']

            if 'start_date' not in stock_params:
                list_date = stock.get('list_date', '20050101')
                stock_params['start_date'] = list_date
            if 'end_date' not in stock_params:
                stock_params['end_date'] = datetime.now().strftime('%Y%m%d')

            task = {
                'func': downloader.download_single_stock,
                'args': (interface_config, stock_params),
                'kwargs': {}
            }
            tasks.append(task)

        results = scheduler.submit_tasks(tasks)

        for result in results:
            if result:
                all_data.extend(result)

        logger.info(f"Batch {batch_idx + 1}/{total_batches} completed. Total records: {len(all_data)}")

    if all_data:
        df = processor.process_data(all_data, interface_config)
        validation_result = processor.validate_data(df, interface_config)
        storage_manager.save_data(interface_name, df.to_dict('records'), async_write=True)
        logger.info(f"Saved total {len(df)} records for {interface_name}")

    return all_data
```

**修改后**（添加文件锁）：
```python
def set(self, key: str, data: Any, ttl: Optional[int] = None) -> bool:
    if ttl is None:
        ttl = self.default_ttl

    cache_path = self._get_cache_path(key)
    temp_path = cache_path + '.tmp'

    try:
        if isinstance(data, list) and len(data) > 0:
            df = pl.DataFrame(data)
            df.write_parquet(temp_path)
        elif isinstance(data, pd.DataFrame):
            df = pl.from_pandas(data)
            df.write_parquet(temp_path)
        elif isinstance(data, pl.DataFrame):
            data.write_parquet(temp_path)
        else:
            logger.warning(f"Cannot cache data of type {type(data)} for key {key}")
            return False

        os.replace(temp_path, cache_path)
        logger.debug(f"Cache set for key: {key}")
        return True
    except Exception as e:
        logger.error(f"Error setting cache for key {key}: {str(e)}")
        if os.path.exists(temp_path):
            try:
                os.remove(temp_path)
            except:
                pass
        return False
```

## 风险和注意事项

### 1. API 限流

**风险**：
- 并发下载可能触发 API 限流
- 高并发可能导致 IP 被封禁

**缓解措施**：
- 控制并发数（建议 4-8）
- 设置合理的速率限制
- 已有 `RateLimiter` 类会自动控制请求速率

### 2. 缓存一致性

**风险**：
- 缓存数据可能与实时数据不一致
- 历史数据可能被修正

**缓解措施**：
- 设置合理的 TTL（24小时）
- 提供手动清空缓存的命令
- 提供 `--force-refresh` 参数强制刷新

### 3. 磁盘空间

**风险**：
- 缓存数据占用大量磁盘空间
- 5470 只股票 × 8000 条记录 × 100 字节 ≈ 4.4 GB

**缓解措施**：
- 定期清理过期缓存
- 使用 `cache_manager.clear_expired()` 命令
- Parquet 格式已压缩

### 4. 线程安全

**风险**：
- 多线程并发写入缓存可能导致数据损坏

**缓解措施**：
- 使用临时文件 + 原子重命名（已实现）
- 避免并发写入同一个缓存键
