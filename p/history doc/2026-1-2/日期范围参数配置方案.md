# 日期范围参数配置方案

## 背景说明

当前的 aspipe_v4 系统中，income_vip 接口的 `start_date` 和 `end_date` 参数默认控制公告日期（ann_date）范围，而非报告期（end_date）范围。但在某些应用场景下，用户需要直接指定报告期范围进行查询。

## 目标

使系统能够通过配置文件（YAML）灵活指定 `start_date` 和 `end_date` 参数的行为：
- 支持按公告日期范围查询
- 支持按报告期范围查询

## 实施方案

### 1. 修改接口配置文件

### 2. 日期范围类型说明

新增参数 `date_parameter_type` 支持以下值：

- `announcement`: `start_date`/`end_date` 控制公告日期范围（现有行为）
- `report_period`: `start_date`/`end_date` 控制报告期范围（新需求行为）

### 3. 实现逻辑

修改 `app4/core/downloader.py` 中的 `_execute_date_range_pagination` 方法，使其能够根据配置文件中的 `date_parameter_type` 来决定如何处理日期参数：

1. 当 `date_parameter_type: "report_period"` 时：
   - 将 `start_date` 和 `end_date` 参数转换为多个 `period` 参数请求
   - 例如，20230401-20230630 范围内的报告期可能包括 20230331、20221231 等

2. 当 `date_parameter_type: "announcement"` 时：
   - 保持现有行为，`start_date`/`end_date` 控制公告日期范围

### 4. 使用方式

配置为报告期范围后，命令行使用方式不变：

```bash
python app4/main.py --start_date 20230401 --end_date 20230630 --interface income_vip
```

但此时参数含义变为：获取报告期在 20230401-20230630 范围内的数据（即获取报告期为 20230331 的数据）。

### 5. 向后兼容性

- 保持现有命令行接口不变
- 默认行为可保持为公告日期范围，以维持向后兼容
- 通过配置文件灵活控制行为
