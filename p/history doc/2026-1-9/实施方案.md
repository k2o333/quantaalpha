# 接口股票循环模式实施方案

## 项目信息
- **项目**: aspipe_v4 融合重构版 (App4)
- **日期**: 2026-01-09
- **目标**: 将14个接口配置为使用股票代码轮询模式

---

## 需求分析

### 需求描述
1. **批量下载模式**：使用 `--tscode-historical` 参数时，批量下载14个接口
   - 下载方式：股票代码轮询
   - 与现有的4个接口有部分重叠

2. **单独下载模式**：使用 `--interface xxx` 参数单独下载某个接口时
   - 下载方式：同样使用股票代码轮询

### 接口列表（共14个）

| 序号 | 接口名称 | 说明 | 现有接口 |
|------|----------|------|----------|
| 1 | stk_rewards | 管理层薪酬和持股 | ✅ 是 |
| 2 | income_vip | 利润表 | ❌ 否 |
| 3 | balancesheet_vip | 资产负债表 | ❌ 否 |
| 4 | cashflow_vip | 现金流量表 | ❌ 否 |

- 最终 tscode_historical 组将包含这14个接口

---

## 方案设计

### 核心思路
1. **配置驱动**：在接口配置中增加标识字段，标记哪些接口使用股票循环模式
2. **参数扩展**：将 `--tscode-historical` 参数指向一个配置组，而不是硬编码的接口列表
3. **动态判断**：在下载逻辑中，根据接口配置动态判断是否使用股票循环模式

### 架构优势
1. **配置驱动**：通过配置文件管理接口列表，易于维护和扩展
2. **统一逻辑**：无论是 `--tscode-historical` 还是 `--interface`，都根据接口配置自动选择下载模式
3. **向后兼容**：不影响现有的其他接口和参数
4. **易于扩展**：未来如需添加接口，只需修改配置文件，无需改动代码

#!/usr/bin/env python3
"""
批量修改接口配置文件，启用 stock_loop 模式
"""
import os
import yaml
from pathlib import Path

# 接口配置文件列表（14个）
INTERFACE_FILES = [
    "stk_rewards.yaml",
    "income_vip.yaml",
    "balancesheet_vip.yaml",
    "cashflow_vip.yaml",
    "forecast_vip.yaml",
    "express_vip.yaml",
    "fina_indicator_vip.yaml",
    "fina_audit.yaml",
    "fina_mainbz_vip.yaml",
    "disclosure_date.yaml",
    "top10_floatholders.yaml",
    "top10_holders.yaml",
    "pledge_stat.yaml",
    "pledge_detail.yaml",
]

# 配置文件路径
CONFIG_DIR = Path("/home/quan/testdata/aspipe_v4/app4/config/interfaces")
BACKUP_DIR = Path("/home/quan/testdata/aspipe_v4/p/2026-1-9/backup")

# 创建备份目录
BACKUP_DIR.mkdir(parents=True, exist_ok=True)

# 修改计数器
success_count = 0
skip_count = 0
error_count = 0

# 遍历所有接口配置文件
for file_name in INTERFACE_FILES:
    file_path = CONFIG_DIR / file_name
    backup_path = BACKUP_DIR / file_name

    # 检查文件是否存在
    if not file_path.exists():
        print(f"✗ 文件不存在: {file_name}")
        error_count += 1
        continue

    # 备份原文件
    import shutil
    shutil.copy(file_path, backup_path)

    try:
        # 读取配置文件
        with open(file_path, 'r', encoding='utf-8') as f:
            config = yaml.safe_load(f)

        # 检查是否已经是 stock_loop 模式
        pagination = config.get('pagination', {})
        if pagination.get('enabled', False) and pagination.get('mode') == 'stock_loop':
            print(f"⊘ 已跳过（已是 stock_loop 模式）: {file_name}")
            skip_count += 1
            continue

        # 修改配置
        config['pagination'] = {
            'enabled': True,
            'mode': 'stock_loop'
        }

        # 写回文件
        with open(file_path, 'w', encoding='utf-8') as f:
            yaml.dump(config, f, default_flow_style=False, allow_unicode=True, sort_keys=False)

        print(f"✓ 修改成功: {file_name}")
        success_count += 1
    except Exception as e:
        print(f"✗ 修改失败: {file_name} - {e}")
        error_count += 1

print(f"\n修改完成: 成功 {success_count}, 跳过 {skip_count}, 失败 {error_count}")
print(f"备份目录: {BACKUP_DIR}")
```

                            month_params = {'month': month}
                            if 'ts_code' in params:
                                month_params['ts_code'] = params['ts_code']
                            data = downloader.download(interface_name, month_params)
                            if data:
                                all_data.extend(data)

                        if all_data:
                            logger.info(f"Successfully downloaded {len(all_data)} records for {interface_name}")
                            process_and_save_data(all_data, interface_name, interface_config, processor, storage_manager)
                        else:
                            logger.warning(f"No data downloaded for {interface_name}")
                        continue  # 跳过普通下载逻辑

                    # 使用接口配置的 rate_limit，如果没有则使用全局默认
                    interface_rate_limit = interface_config.get('permissions', {}).get('rate_limit', global_rate_limit)
                    interface_rate_limiter = RateLimiter(interface_rate_limit)

                    # 等待获取令牌
                    interface_rate_limiter.wait_for_tokens(1)

                    # 下载数据
                    data = downloader.download(interface_name, params)

                    if data:
                        logger.info(f"Successfully downloaded {len(data)} records for {interface_name}")
                        process_and_save_data(data, interface_name, interface_config, processor, storage_manager)
                    else:
                        logger.warning(f"No data downloaded for {interface_name}")

            except Exception as e:
                logger.error(f"Error processing interface {interface_name}: {str(e)}")
```

## 注意事项

### 1. 并发控制
- 股票循环模式会并发下载多只股票的数据
- 需要根据 API 限制调整并发数
- 当前默认并发数为 4（可在 settings.yaml 中修改）

### 2. 数据量预估
- 假设有 5000 只股票，每个接口平均 1000 条数据
- 14 个接口总数据量约：5000 * 1000 * 14 = 7000 万条
- 需要确保磁盘空间充足

### 3. API 限制
- 注意 TuShare API 的积分限制和请求频率限制
- 建议在非高峰期执行大批量下载

### 4. 错误处理
- 如果某个接口下载失败，会跳过该接口继续处理下一个
- 失败的接口需要重新运行或单独下载

### 5. 代码修改注意事项
- Python 对缩进敏感，确保替换后的代码缩进正确（使用4个空格）
- 修改前务必备份原始 `main.py` 文件
- 建议先测试单个接口，再测试批量下载
