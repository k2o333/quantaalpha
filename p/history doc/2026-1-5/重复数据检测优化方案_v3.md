## 1. 核心改进 (vs v2)

本方案在 v2 的**窗口级检测**基础上，融合了代码评审中的高价值建议，升级为**v3 版本**。主要改进点如下：

1.  **配置驱动 (Config Driven)**：不再硬编码 `trade_date`，支持通过配置或智能推断确定检测列（兼容 `cal_date`, `end_date`, `period` 等）。
2.  **安全降级 (Fail-Safe)**：增加完善的异常处理，确保存储读取失败时自动回退到下载模式，不中断任务。
3.  **多模式支持 (Multi-Mode)**：不仅优化 `date_range`，同时将检测逻辑扩展至 `period_range`（报告期）模式。

## 3. 核心检测逻辑实现

### A. 智能列名解析

```python
def _determine_detection_column(self, interface_config: Dict[str, Any]) -> Optional[str]:
    # 1. 优先读取接口配置中的显式定义
    # 在 interface.yaml 中可配置: duplicate_detection: { date_column: 'ann_date' }
    config_col = interface_config.get('duplicate_detection', {}).get('date_column')
    if config_col:
        return config_col

    # 2. 尝试从 parameters 中推断 (如 start_date/end_date 对应的字段)
    # 略...

    # 3. 常见字段启发式匹配
    potential_cols = ['trade_date', 'cal_date', 'end_date', 'ann_date', 'datetime']
    # 注意：这里无法直接校验字段是否存在，需要依赖接口文档或试错。
    # 更稳妥的方式是默认 'trade_date'，或者由上层逻辑传入。
    return 'trade_date'
```

### B. 通用检测方法 (`_check_coverage`)

```python
def _check_coverage(self, interface_name: str, start_date: str, end_date: str, date_col: str) -> bool:
    """
    通用覆盖率检测
    返回: True (已覆盖, 跳过下载) / False (未覆盖, 需要下载)
    """
    if not self.storage_manager:
        return False

    try:
        # [关键性能点] 1. 列投影：只读取时间列
        # [关键性能点] 2. 文件名过滤：StorageManager 会根据 start_date/end_date 预筛选文件
        df = self.storage_manager.read_interface_data(
            interface_name, 
            start_date=start_date, 
            end_date=end_date,
            columns=[date_col]
        )
        
        if df.is_empty():
            return False

        # 3. 获取理论上的时间点数量 (以交易日历为基准)
        # 注意：对于非交易日接口（如财报），这种比较可能不准确，需根据模式区分
        trade_cal = self._get_trade_calendar_from_data_dir(start_date, end_date)
        if not trade_cal:
            # 如果获取不到日历，无法计算覆盖率，保守起见返回 False
            return False
            
        expected_count = len([d for d in trade_cal if d.get('is_open') == 1])
        actual_count = df.select(pl.col(date_col)).n_unique()

        # 4. 判定逻辑：覆盖率 >= 95%
        if expected_count == 0: return False
        coverage = actual_count / expected_count
        return coverage >= 0.95

    except Exception as e:
        # [安全降级] 发生任何错误（如IO错误、列不存在），记录警告并要求重新下载
        logger.warning(f"Coverage check failed for {interface_name}: {e}. Fallback to download.")
        return False
```

## 4. 嵌入下载工作流

### 场景 1: 日期范围 (`date_range`)
在 `_execute_date_range_pagination` 的窗口循环中调用。

```python

window_end = ...

# 1. 确定检测列

# 2. 执行检测
if self._check_coverage(interface_config['api_name'], window_start, window_end, date_col):
    logger.info(f"Skipping window {window_start}-{window_end} (Data exists)")
    continue

# 3. 下载
# ...
```

### 场景 2: 报告期 (`period_range`)
在 `_execute_period_range_pagination` 中，针对每个 `period` 进行精确检测。

```python
# ... Inside period loop ...
for period in periods:
    # 1. 检测该 period 是否已存在
    # 特殊逻辑：直接查询 storage 是否包含该 period
    try:
        df = self.storage_manager.read_interface_data(
            interface_config['api_name'],
            columns=['period']  # 财报接口通常主键为 period + ts_code
        )
        if not df.is_empty() and period in df['period'].unique().to_list():
             # 还需要检查 ts_code 覆盖率吗？
             # 简化版：如果该 period 有数据，且数据量足够大（例如 > 4000条），则认为已下载
             count = df.filter(pl.col('period') == period).height
             if count > 1000: # 阈值可配置
                 logger.info(f"Skipping period {period} (Data exists: {count} records)")
                 continue
    except Exception:
        pass # Fallback

    # 2. 下载
    # ...
```

## 5. 配置化扩展示例

建议在 `config/interfaces/xxx.yaml` 中支持如下配置：

```yaml
# 示例：日线行情
api_name: daily
duplicate_detection:
  enabled: true
  date_column: trade_date
  threshold: 0.95

# 示例：财报
api_name: income_vip
duplicate_detection:
  enabled: true
  mode: exact_match  # 精确匹配 period
  date_column: period
```
