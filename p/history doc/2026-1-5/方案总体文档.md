# Cache 改为 Data 目录去重方案

**项目**: aspipe_v4/app4
**日期**: 2026-01-05

---

## 目标

去掉 cache 方案，每次下载前把已有数据的 primary key 加载到内存，下载时对比去重，然后写入 data 目录。

---

## 设计思路

```
下载前: 读取 parquet 文件的 primary key → 内存
         ↓
下载中: 获取 API 数据 → 对比 primary key → 过滤重复 → 批次内去重
         ↓
写入: 异步写入 parquet（保持现有机制）
```

**关键点**:
1. 下载前只加载 primary key 列到内存（不加载全部数据）
2. 下载时对比，重复的记录直接丢弃
3. 同一批次内的数据也要去重
4. 写入时直接追加（因为下载时已经去重了）

### 1. `core/storage.py` - 新增方法

添加一个方法，用于读取已有数据的 primary key：

```python
def load_existing_keys(self, interface_name: str) -> set:
    """
    读取已有数据的主键到内存

    Args:
        interface_name: 接口名称

    Returns:
        主键集合
    """
    file_path = os.path.join(self.storage_dir, f"{interface_name}.{self.format}")

    if not os.path.exists(file_path) or self.format != "parquet":
        return set()

    try:
        # 获取接口配置
        interface_config = self._get_interface_config(interface_name)
        primary_key = interface_config.get('output', {}).get('primary_key', [])

        if not primary_key:
            return set()

        # 只读取主键列（节省内存）
        df = pl.read_parquet(file_path, columns=primary_key)

        # 将主键转换为元组集合
        key_tuples = df.select(primary_key).to_tuples()

        return set(key_tuples)

    except Exception as e:
        logger.error(f"Error loading existing keys for {interface_name}: {str(e)}")
        return set()
```

### 2. `core/processor.py` - 去重逻辑

修改 `process_data` 方法，添加去重参数：

```python
def process_data(self, data: List[Dict[str, Any]], interface_config: Dict[str, Any],
                  existing_keys: set = None) -> pl.DataFrame:
    """
    处理数据 - 包含去重

    Args:
        data: 原始数据列表
        interface_config: 接口配置
        existing_keys: 已有数据的主键集合（可选）

    Returns:
        处理后的 DataFrame
    """
    if not data:
        return pl.DataFrame()

    try:
        interface_name = interface_config.get('api_name', 'unknown')

        # 创建 DataFrame
        df = SchemaManager.create_dataframe(data, interface_name)
        if df.is_empty() and data:
            df = pl.DataFrame(data, infer_schema_length=len(data))

        df.columns = [str(col) for col in df.columns]
        df = df.select([col for i, col in enumerate(df.columns) if col not in df.columns[:i]])

        # 应用类型转换
        df = self._apply_type_conversions(df, interface_config)

        # [新增] 对比已有数据去重
        if existing_keys:
            output_config = interface_config.get('output', {})
            primary_keys = output_config.get('primary_key', [])
            existing_key_fields = [key for key in primary_keys if key in df.columns]

            if existing_key_fields:
                # 过滤掉已有的记录
                before_filter = len(df)
                df = df.filter(
                    ~pl.struct(existing_key_fields).is_in(
                        [tuple(k) for k in existing_keys]
                    )
                )
                after_filter = len(df)
                if before_filter > after_filter:
                    logger.info(f"Filtered out {before_filter - after_filter} existing records for {interface_name}")

        # [新增] 批次内去重
        output_config = interface_config.get('output', {})
        primary_keys = output_config.get('primary_key', [])
        existing_key_fields = [key for key in primary_keys if key in df.columns]

        if existing_key_fields:
            before_dedup = len(df)
            df = df.unique(subset=existing_key_fields, keep='last')
            after_dedup = len(df)
            if before_dedup > after_dedup:
                logger.info(f"Removed {before_dedup - after_dedup} duplicates in current batch for {interface_name}")

        # 检查和处理主键
        df = self._handle_primary_keys(df, interface_config)

        # 数据清洗
        df = self._clean_data(df, interface_config)

        logger.info(f"Processed {len(df)} records for {interface_name}")
        return df

    except Exception as e:
        logger.error(f"Error processing data for {interface_name}: {str(e)}")
        return pl.DataFrame()
```

### 3. `main.py` - 修改下载流程

修改 `process_and_save_data` 函数和下载逻辑：

```python
def process_and_save_data(data, interface_name, interface_config, processor, storage_manager, existing_keys=None):
    """处理并保存数据"""
    if not data:
        logger.warning(f"No data to process for {interface_name}")
        return None

    # 处理数据（传入已有主键）
    df = processor.process_data(data, interface_config, existing_keys=existing_keys)
    validation_result = processor.validate_data(df, interface_config)

    # 保存数据
    if not df.is_empty():
        storage_manager.save_data(interface_name, df.to_dicts(), async_write=True)
        logger.info(f"Saved {len(df)} processed records for {interface_name}")
    else:
        logger.info(f"No new records to save for {interface_name}")

    return df
```

## 优势

1. **性能好**: 只加载 primary key 列，内存占用小
2. **逻辑清晰**: 下载时去重，写入时直接追加
3. **改动小**: 不需要修改写入逻辑，不需要文件锁
4. **支持并发**: 每个进程加载自己的 primary key，互不影响

---

## 注意事项

1. **并发问题**: 如果多个进程同时下载同一个接口，可能会重复数据。建议：
   - 使用一个主进程下载
   - 或者增加文件锁保护写入

2. **Primary Key 配置**: 确保每个接口的配置文件中都定义了 `primary_key`

3. **内存占用**: 对于超大数据量（>1000万条），primary key 占用约 1-2GB，需要注意内存

## 示例流程

```
1. 下载前:
   storage_manager.load_existing_keys('daily')
   → 读取 data/daily.parquet 的 ts_code, trade_date 列
   → 返回 set(('000001.SZ', '20240101'), ('000001.SZ', '20240102'), ...)

2. 下载 API 数据:
   获取到 100 条记录

3. 对比去重:
   80 条在已有主键中 → 丢弃
   20 条新数据 → 保留

4. 批次内去重:
   发现 2 条重复（相同的 ts_code + trade_date）
   保留 18 条

5. 写入:
   将 18 条新数据追加到 data/daily.parquet
```
