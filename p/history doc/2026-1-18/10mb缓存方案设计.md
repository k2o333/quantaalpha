# 10MB缓存方案设计文档

**日期**：2026-01-20  
**需求**：每个接口数据累积到10MB后去重并保存到Parquet文件，每个接口单worker执行

---

## 一、需求分析

### 核心要求
1. **内存暂存**：数据在内存中累积，达到10MB后才处理（速度快）
2. **单Worker机制**：每个接口只分配一个worker，顺序处理所有任务
3. **自动去重**：利用已有`process_and_save_data`函数的去重功能
4. **Parquet存储**：每个接口一个目录，文件自动追加或新建

### 现有架构分析
- **任务调度**：`TaskScheduler`使用ThreadPoolExecutor，默认并发执行
- **存储机制**：`StorageManager`有异步写入线程，但暂存逻辑在main.py
- **数据处理**：`process_and_save_data`函数已包含完整去重逻辑
- **股票循环**：`run_concurrent_stock_download`函数目前是多线程并发下载

## 二、方案对比

### 方案1：StorageManager层缓存（不推荐）
**实现位置**：`core/storage.py`

**优点**：
- 改动集中，不分散到业务逻辑
- 自动利用异步写入机制

**缺点**：
- 数据已放入队列，10MB计算不准确
- 无法精确控制单worker（调度层已分发任务）
- 需要修改StorageManager核心逻辑

**结论**：不符合"单worker"和"内存暂存"的核心需求

**实现位置**：`app4/main.py`的`run_concurrent_stock_download`函数

- 单个worker顺序遍历股票列表
- 内存中累积数据（`all_data`列表）
- 达到10MB阈值时调用`process_and_save_data`

**改动点**：
```

├── 新增：estimate_data_size() 函数
├── 新增：run_single_interface_download() 函数（60行）
└── 修改：约3处调用点（stock_loop模式）
```

**优点**：
- ✅ 内存暂存，速度快
- ✅ 单worker，逻辑简单清晰
- ✅ 利用现有去重机制
- ✅ 改动集中，约75行代码
- ✅ 不修改Scheduler/Downloader/StorageManager

**缺点**：
- 需重构现有并发下载函数
- 单worker下载速度可能慢于并发（但符合需求）

**实现位置**：`app4/main.py`的`process_and_save_data`函数

- 保持并发下载（多worker）

## 三、推荐方案：方案2详细设计

### 3.1 新增函数1：数据大小估算
```python
def estimate_data_size(data: List[Dict[str, Any]]) -> int:
    """
    估算数据列表的字节大小

    估算规则：
    - 每条记录约500字节（根据实际数据结构调整）
    - 可根据字段数量和类型动态调整

    Args:
        data: 数据列表

    Returns:
        估算的字节数
    """
    if not data:
        return 0

    # 基础估算：每条记录500字节
    base_size = len(data) * 500

    # 如果有数据样本，进行更精确的估算
    # sample = data[0]
    # field_count = len(sample)
    # base_size = len(data) * field_count * 50  # 每个字段约50字节

    return base_size
```

**位置**：`app4/main.py`顶部，与其他工具函数并列

### 3.2 新增函数2：单接口下载（核心逻辑）
```python
def run_single_interface_download(
    downloader,
    interface_name: str,
    interface_config: Dict[str, Any],
    base_params: Dict[str, Any],
    stock_list: List[Dict[str, Any]],
    rate_limiter,
    storage_manager,
    processor
) -> int:
    """
    单worker下载整个接口的数据，累积到10MB后处理

    Args:
        downloader: 下载器实例
        interface_name: 接口名称
        interface_config: 接口配置
        base_params: 基础请求参数
        stock_list: 股票列表
        rate_limiter: 速率限制器
        storage_manager: 存储管理器
        processor: 数据处理器

    Returns:
        处理的总记录数
    """
    all_data = []      # 内存中累积数据的列表
    size_bytes = 0     # 当前累积的数据大小（字节）
    total_records = 0  # 总记录数统计

    logger.info(f"Starting single-worker download for {interface_name}, "
                f"{len(stock_list)} stocks, threshold: 10MB")

    # 遍历所有股票（单worker顺序处理）
    for i, stock in enumerate(stock_list):
        ts_code = stock['ts_code']

        # 1. 速率限制
        rate_limiter.wait_for_tokens(1)

        # 2. 下载单只股票数据
        stock_data = downloader.download_single_stock(
            interface_config, stock, base_params
        )

        # 3. 数据累积
        if stock_data:
            all_data.extend(stock_data)
            size_bytes += estimate_data_size(stock_data)
            total_records += len(stock_data)

        # 4. 进度日志（每100只）
        if (i + 1) % 100 == 0:
            logger.info(f"Progress: {i + 1}/{len(stock_list)} stocks, "
                        f"buffer: {len(all_data)} records, "
                        f"size: {size_bytes / 1024 / 1024:.2f}MB")

        # 5. 达到阈值或最后一只股票时处理
        if size_bytes >= 10 * 1024 * 1024 or i == len(stock_list) - 1:
            if all_data:
                logger.info(f"Flushing buffer for {interface_name}: "
                            f"{len(all_data)} records, "
                            f"{size_bytes / 1024 / 1024:.2f}MB")

                # 6. 处理并保存（包含去重逻辑）
                process_and_save_data(
                    all_data, interface_name, interface_config,
                    processor, storage_manager
                )

                # 7. 清空缓存
                all_data = []
                size_bytes = 0

    logger.info(f"Completed download for {interface_name}, "
                f"total {total_records} records")
    return total_records
```

**位置**：`app4/main.py`，替换现有的`run_concurrent_stock_download`函数

## 四、工作流程详解

### 场景示例：下载`pro_bar`接口，5000只股票

```
开始 → 股票1 → 股票2 → ... → 股票50
  ↓
累积数据: 1.2MB → 2.4MB → ... → 10.5MB（触发阈值）
  ↓
process_and_save_data() 处理前50只股票数据
  ↓
清空缓存
  ↓
继续: 股票51 → 股票52 → ... → 股票100
  ↓
累积数据: 0.3MB → 0.6MB → ... → 10.2MB（触发阈值）
  ↓
process_and_save_data() 处理股票51-100数据
  ↓
清空缓存
  ↓
...
  ↓
最后: 股票4951-5000（可能不足10MB）
  ↓
process_and_save_data() 处理剩余数据
  ↓
结束
```

### 数据流向
```
API响应 → downloader.download_single_stock() → all_data列表（内存）
  ↓（达到10MB或最后一只股票）
process_and_save_data() → 去重处理 → storage_manager.save_data()
  ↓
Parquet文件（单个接口目录下）
```

### 去重机制
```python
# 在process_and_save_data()中已实现：

1. 读取接口现有Parquet文件（如果存在）
   existing_df = storage_manager.read_interface_data(interface_name, columns=dedup_columns)

2. 构建现有主键集合（Set去重，O(1)查询）
   existing_keys = {(ts_code, trade_date), (ts_code, trade_date), ...}

3. 过滤新数据
   new_records = [record for record in all_data if key not in existing_keys]

4. 保存新数据到Parquet
   storage_manager.save_data(interface_name, new_records, async_write=True)
```

## 六、方案优缺点

### 优点

1. **完全符合需求**
   - ✅ 内存暂存：数据在`all_data`列表中累积
   - ✅ 单worker：顺序遍历股票列表
   - ✅ 自动去重：复用`process_and_save_data`
   - ✅ 10MB阈值：精确控制触发时机

2. **性能平衡**
   - 减少小文件数量（每10MB一个文件，而非每只股票一个）
   - 内存操作速度快
   - 异步写入不阻塞下载

3. **改动最小化**
   - 仅修改`main.py`约90行
   - 不破坏现有架构
   - 其他组件无需改动

4. **可调试性强**
   - 单worker日志清晰（按顺序处理）
   - 每个批次有明确的日志输出
   - 易于定位问题

5. **灵活性高**
   - 阈值可调（修改`10 * 1024 * 1024`）
   - 估算算法可优化（根据实际数据特征）
   - 保留多worker能力（如需可快速切换回并发版本）

### 缺点

1. **下载速度可能较慢**
   - 单worker无法充分利用多核CPU
   - 网络IO等待时CPU空闲

2. **内存占用风险**
   - 极端情况：单只股票数据 > 10MB
   - 累积数据可能超过10MB阈值

3. **故障恢复能力弱**
   - 内存中数据在程序崩溃时丢失
   - 需要重新下载整个批次

4. **估算精度问题**
   - `estimate_data_size()`是估算值，可能偏差
   - 需要根据实际情况调整估算系数

## 七、优化建议

### 优化1：精确估算数据大小
```python
def estimate_data_size(data: List[Dict[str, Any]]) -> int:
    """精确估算数据大小"""
    if not data:
        return 0

    # 采样第一条记录
    sample = data[0]
    field_count = len(sample)

    # 根据字段类型估算
    size_per_record = 0
    for key, value in sample.items():
        if isinstance(value, str):
            size_per_record += len(value)  # 字符串按实际长度
        elif isinstance(value, (int, float)):
            size_per_record += 8  # 数字按8字节
        elif isinstance(value, bool):
            size_per_record += 1
        else:
            size_per_record += 50  # 其他类型默认50字节

    # 总大小 = 单条大小 * 记录数
    return len(data) * size_per_record
```

### 优化2：内存保护机制
```python
# 在run_single_interface_download中增加保护
```
