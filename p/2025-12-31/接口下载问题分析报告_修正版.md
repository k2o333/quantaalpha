## 二、发现的问题

### 2.1 严重问题：broker_recommend接口参数错误

**错误信息**:
```
API error for broker_recommend: 必填参数, month
```

**原因分析**:
- broker_recommend接口需要`month`参数（格式：YYYYMM）
- 但系统只传递了`start_date`和`end_date`参数
- 该接口的配置文件显示`month`是必填参数

**配置文件**: [broker_recommend.yaml](file:///home/quan/testdata/aspipe_v4/app4/config/interfaces/broker_recommend.yaml#L17)
```yaml
parameters:
  month:
    type: string
    required: true  # 必填参数
    description: "月度 YYYYMM"
```

**影响**: 该接口无法正常下载数据

**建议修复方案**:
1. 方案A：修改main.py，为broker_recommend接口生成正确的month参数
2. 方案B：在默认接口列表中排除broker_recommend接口
3. 方案C：添加参数映射逻辑，将start_date/end_date转换为month列表

### 2.2 严重问题：stock_hsgt接口过滤逻辑错误

**问题分析**:
- stock_hsgt接口需要`type`参数（必填），类型为HK_SZ/SZ_HK/HK_SH/SH_HK
- 系统将此接口过滤掉了，但实际应该根据type参数分别请求
- 该接口返回的是沪深港通股票列表，不是单个股票的数据

**影响**: 丢失了重要的沪深港通数据

**建议修复方案**:
1. 修改过滤逻辑，对stock_hsgt接口按type参数分别请求
2. 或者在配置文件中指定默认type值

### 2.3 中等问题：API数据量截断警告

**警告信息**:
```
Window 20250630-20251231 returned 6000 records, which may be truncated (API limit: 2000)
```

**涉及接口**: cyq_chips（每日筹码分布）

**原因分析**:
- API单次请求限制为2000条记录
- 实际返回了6000条记录，说明数据被截断
- 去重后只剩57条有效记录

**配置文件**: [cyq_chips.yaml](file:///home/quan/testdata/aspipe_v4/app4/config/interfaces/cyq_chips.yaml#L9)
```yaml
permissions:
  query_limit: 2000  # API限制
```

**影响**: 可能丢失部分数据

**建议修复方案**:
1. 减小window_size_days（当前为365天）
2. 实现更细粒度的分页逻辑
3. 添加数据完整性检查，检测截断情况
4. 考虑使用stock_loop模式逐股票下载

## 三、根本原因分析

### 3.1 为什么很多接口没有数据？

1. **日期范围问题**
   - 使用了未来日期（2025年12月31日）
   - 很多接口在未来日期没有数据
   - 财务数据需要等到财报发布后才有数据

2. **接口特性**
   - 事件类接口（如namechange、repurchase）在指定期间没有发生
   - 状态类接口（如stock_st、suspend_d）不适用于该股票
   - 数据类接口（如express、forecast）在指定期间没有发布

3. **参数不匹配**
   - broker_recommend需要month参数，但系统传递的是日期范围
   - stock_hsgt需要type参数，但被系统过滤

### 3.2 接口分类
