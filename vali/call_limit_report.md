# TuShare API 单次调用记录数限制报告

## 概述
本报告详细分析了5000积分token在TuShare API各接口上的单次调用最大返回记录数限制。

## 测试环境
- Token积分: 5000
- 测试时间: 2025-12-09
- 代理设置: 已启用 (http://tushare.xyz:5000)

## 单次调用记录数限制详情

### 1. 支持limit参数的接口

#### stock_basic
- **调用方式**: `pro.stock_basic(exchange='', list_status='L', limit=X)`
- **单次最大记录数**: 无严格限制（测试中limit=5000返回5000条）
- **说明**: 实际返回记录数取决于全市场股票数量，目前约为5500+只
- **建议limit值**: 1000-5000

#### income_vip (VIP接口)
- **调用方式**: `pro.income_vip(period='20240331', limit=X)`
- **单次最大记录数**: 无严格限制（测试中limit=5000返回5000条）
- **说明**: 实际返回记录数取决于单季度财务报告数量
- **建议limit值**: 1000-5000

#### balancesheet_vip (VIP接口)
- **调用方式**: `pro.balancesheet_vip(period='20240331', limit=X)`
- **单次最大记录数**: 无严格限制
- **说明**: 实际返回记录数取决于单季度财务报告数量

#### cashflow_vip (VIP接口)
- **调用方式**: `pro.cashflow_vip(period='20240331', limit=X)`
- **单次最大记录数**: 无严格限制
- **说明**: 实际返回记录数取决于单季度财务报告数量

#### fina_indicator_vip (VIP接口)
- **调用方式**: `pro.fina_indicator_vip(period='20240331', limit=X)`
- **单次最大记录数**: 无严格限制
- **说明**: 实际返回记录数取决于单季度财务报告数量

#### stk_surv
- **调用方式**: `pro.stk_surv(limit=X)`
- **单次最大记录数**: 400条
- **说明**: 测试发现即使设置limit=500，也只能获取400条记录
- **建议limit值**: 400

### 2. 固定返回记录数的接口

#### daily_basic
- **调用方式**: `pro.daily_basic(trade_date='YYYYMMDD')`
- **单次固定记录数**: 约5362条记录（单日全市场数据）
- **说明**: 返回指定交易日所有股票的基本面指标
- **无法通过参数控制**: 返回记录数由当日市场股票数量决定

#### moneyflow
- **调用方式**: `pro.moneyflow(trade_date='YYYYMMDD')`
- **单次固定记录数**: 约5103条记录（单日全市场数据）
- **说明**: 返回指定交易日所有股票的资金流向数据
- **无法通过参数控制**: 返回记录数由当日市场股票数量决定

#### top10_holders
- **调用方式**: `pro.top10_holders(ts_code='XXXXXX.XX', period='YYYYMMDD')`
- **单次固定记录数**: 最多10条记录
- **说明**: 每个股票每个报告期最多返回10大股东
- **无法通过参数控制**: 返回记录数固定

#### stk_factor
- **调用方式**: `pro.stk_factor(ts_code='XXXXXX.XX', trade_date='YYYYMMDD')`
- **单次固定记录数**: 1条记录
- **说明**: 每个股票每个交易日返回1条技术因子数据
- **无法通过参数控制**: 返回记录数固定

### 3. 时间范围查询接口

#### trade_cal
- **调用方式**: `pro.trade_cal(exchange='SSE', start_date='YYYYMMDD', end_date='YYYYMMDD')`
- **单次最大记录数**: 取决于时间范围
- **说明**: 返回指定交易所和时间范围内的交易日历
- **实际限制**: 通常1年约365条，10年约3650条

## 关键发现

### 5000积分用户的优势
1. **无limit参数严格限制**: 大部分接口可设置较高的limit值，不像低积分用户有严格限制
2. **支持大批量数据获取**: VIP接口单次可获取数千条全市场数据
3. **更高的单日数据量**: daily_basic、moneyflow等接口可获取全市场数据

### 实际限制情况
1. **真正的硬限制**: `stk_surv`接口存在400条记录的硬限制
2. **数据量依赖性**: 大多数接口的记录数取决于实际数据量，而非API限制
3. **固定数据结构**: 部分接口返回记录数固定，无法通过参数控制

## 最佳实践建议

### 针对不同接口的调用策略
1. **支持limit接口**: 合理设置limit参数（1000-5000），避免一次性请求过多数据
2. **固定记录数接口**: 无需设置limit参数，直接按需求调用
3. **大批量获取**: 优先使用VIP接口，可一次性获取全市场数据

### 性能优化
1. **批量处理**: 对于大批量数据，使用支持limit参数的接口进行分批获取
2. **避免超时**: 不要设置过大的limit值，可能导致API调用超时
3. **数据缓存**: 对于返回记录数固定的接口，可适当缓存数据

## 结论

5000积分token在单次调用记录数限制方面具有显著优势，大部分接口无严格的限制，可一次性获取大量数据，特别是VIP接口可获取全市场数据，大大提升了数据获取效率。
