# CPU/单显卡版：轻量级机器学习软分域（MoE）方案

## 版本：3.0 (算力受限版)
## 适用场景：A股全市场中高频Alpha策略，仅具备单机CPU与中端GPU（如 RTX 2060）
## 核心目标：完全抛弃深度学习和神经网络，仅使用 GMM（高斯混合模型）配合 LightGBM / XGBoost，严密解决“软标签生成”、“相对特征死循环”和“训练目标断层”等所有反馈痛点，构建极低算力开销的 MoE。

---

### 一、 架构重构与逻辑自洽 (回应评审缺陷)

本方案彻底抛弃了“端到端联合训练（Backprop）”的幻想，采用**“软聚类制导 + 样本权重隔离”**的传统机器学习阶段性打法。

#### 1. 门控网络到底是什么？（解决软聚类标签来源）
* **原痛点**：未说明门控标签从哪里来。
* **新方案设计**：**门控网络直接就是 GMM（高斯混合模型）本身**，或者用一个极小的 LightGBM-multiclass 拟合 GMM 的输出。
* **逻辑**：使用低频稳定特征 $X_{gate}$（市值、换手、低频波动）训练 GMM。对于任何一只股票，GMM 能够原生地输出它属于 $K$ 个分布组件的后验概率 $P(C_k|X_{gate})$。
* **妙处**：这套概率天然就是一组合格的门控权重 $[w_1, w_2, ..., w_K]$，且严格满足 $\sum w_i = 1$。不需要瞎造“专家标签”。

#### 2. “不准用硬类，又要算类相对特征”的死结如何解？（解决逻辑矛盾）
* **新方案设计**：彻底废弃“与簇均值比较”的荒谬特征计算法。
* 专家模型的输入 $X_{expert}$ 全部改为**全市场横截面特征**或**行业内横截面特征**：
  * 横向Z-Score：`momentum_rel = (momentum - 全市场当日均值) / 全市场当日标准差`
  * 行业内相对值：`(个股PE - 申万一级行业平均PE)`
* 这样，无论后面怎么分域，特征工程在前面是完全自洽且算力友好的。

#### 3. 专家网络如何训练？（解决训练目标断层）
在没有深度学习梯度反传的情况下，如何让 $K$ 个 LightGBM 各自成为一个“领域专家”？
* **核心武器：样本权重 (Sample Weights)**
* 我们不硬切分数据集。对于专家模型 $E_k$，我们将全市场的全部训练股票喂给它，但给每个样本 $i$ 赋予一个训练权重 $Weight_{i,k} = w_{i,k}$ （即 GMM 吐出的软聚类概率）。
* **效果**：专家 $E_1$ 会重点关注 $w_1$ 很高的“大盘价值股”（因为被惩罚的高权重在那），而对 $w_1$ 接近 0 的“小盘高波股”极其无视。通过修改 Loss 函数的计算配比，强行在同一个全局数据集上，逼出了 $K$ 个专长不同的树模型专家！

### 二、 极简三段式落地流水线

#### Phase 1: 确定专家数量与边界 (GMM 训练)
1. 提取全市场历史5年的 $X_{gate}$ 数据。
2. 跑多次 GMM 聚类。如何定 $K$ 的数量？（回应评审意见）
   * 对 $K=4,5,6,7,8$ 分别跑 GMM。
   * 计算 **BIC（贝叶斯信息准则）** 或 **Silhouette Score（轮廓系数）**，寻找拐点，科学定出专家数量 $K$。
3. 确定最优 GMM 模型，保存模型文件 `gating_gmm.pkl`。这个模型通过 `.predict_proba()` 出来的就是门控权重。

#### Phase 2: 并行训练受限专家 (LightGBM 样本加权)
针对确定好的 $K$（假设 $K=5$），利用单机 CPU 的多线程优势或 2060 显卡：
1. 跑全市场数据，通过 `gating_gmm.pkl` 生成每只股票的 5 个权重 $(w_1, w_2, w_3, w_4, w_5)$。
2. 启动 5 次独立的 LightGBM 训练任务（可以串行跑，极其省内存）：
   * 训练 $Expert_1$ 时，喂入全部数据，`sample_weight = 数据集中的 w_1 列`，设 `device="gpu"` 交给 2060 加速。
   * 训练 $Expert_2$ 时，喂入全部数据，`sample_weight = 数据集中的 w_2 列`...
3. 保存这 5 个 `lgb_expert_1~5.txt` 模型。

#### Phase 3: 每日在线特征与推理平滑 (Inference)
在单日运算中（完全能在老旧机器的一分钟内跑完）：
1. 准备当日的 $X_{gate}$ 与 $X_{expert}$（只需要算全市场均值/标准差）。
2. **门控路由**：`weights = gmm.predict_proba(X_gate)`。
3. **专家打分**：5 个 LightGBM 各自对全市场 predict 吐出 5 组超额预测值 $(y_1, y_2, y_3, y_4, y_5)$。
4. **加权融合**：对于每只股票，执行向量点乘 $Y_{final} = w_1 \cdot y_1 + w_2 \cdot y_2 + ... + w_5 \cdot y_5$。

---

### 三、 增设可信验证 (Baseline & 消融实验)

* **Baseline-0 (全局单模)**：抛弃 GMM 和所有权重，直接用 LightGBM 把全市场数据训一个统管全局的模型。记录换手率与IC。
* **Baseline-1 (硬聚类树)**：使用 K-Means 硬切分数据（不用权重，物理切断），训5个小 LightGBM。用它来重现“簇漂移导致的每天剧烈换手和回撤放大”。
* **本方案验证**：观察本方案的加权融合 $Y_{final}$，是否成功在 Baseline-1 换手率剧减的情况下，找回甚至超越 Baseline-0 的多头超额收益。

### 四、 总体评价本方案针对性
1. **可行性**：⭐⭐⭐⭐⭐ 纯依赖 `scikit-learn` 和 `LightGBM`，2060 充足发挥树模型建树优势，无任何显存/内存溢出危险。
2. **完整性**：⭐⭐⭐⭐⭐ 从特征、训练目标到权重来源形成完美逻辑闭环，无任何“偷窥数据”黑边。
3. **透明可解释**：相比 NN 黑盒，GMM 可以清晰印出哪几组均值和方差对应什么股票类型，LightGBM 可以清晰打出各个类的主导特征 Tree Split Importance。
