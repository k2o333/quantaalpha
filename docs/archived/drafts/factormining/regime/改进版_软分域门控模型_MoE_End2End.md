# 改进版：软分域门控模型（MoE）端到端框架方案

## 版本：2.0
## 适用场景：A股全市场动态量价及基本面Alpha策略 (从零搭建)
## 核心目标：解决硬聚类带来的“簇漂移”问题，实现模型输出平滑过渡，同时消除原方案的逻辑矛盾，构建严谨、可落地的端到端联合训练框架。

---

### 一、 核心架构重构与逻辑修正

针对原方案的逻辑矛盾（既要避免硬标签，又需要知道特定簇的均值），本方案对“特征工程”和“模型选型”进行了彻底重构：

#### 1. 特征工程去“依赖化” (解决逻辑矛盾)
* **原错误逻辑**：输入特征依赖不明晰的“簇均值”（`(个股 - 簇均值) / 簇标准差`）。如果不用硬聚类，这个群体基准无法定义。
* **强化逻辑（基于全市场状态截面化）**：专家模型（Expert）和门控网络（Gating）的输入特征彻底与“人工簇”解绑。
  * **门控网络特征**：低频稳定的风格特征（如：对数市值 `log_mcap`、月均换手率 `turnover_20d`、20日波动率 `vol_20d`）。
  * **专家网络特征**：使用**全市场横截面标准化**的特征。不论股票分在哪个域，它看到的是它在当时全市场中的相对位置。例如：
    * 横向Z-Score：`momentum_rel = (momentum - market_mean) / market_std`
    * 横向百分位排位：`momentum_rank_pct = rank(momentum) / N`

#### 2. 底层模型大统一 (解决训练目标断层)
* **原方案堵点**：利用 LightGBM 作为专家模型，它是一个不可导的树模型，导致“门控网络”的输出无法直接参与梯度回传，变成了生硬的“两张皮”。
* **本方案修正**：为了实现端到端（End-to-End）联合训练，全部**替换为基于神经架构（如 PyTorch）的深度学习模型（MLP 或 Transformer Encoder）**。

#### 3. 定义端到端的网络结构
* **输入层**：分别输入门控特征 $X_{gate}$ 和多因子特征 $X_{expert}$。
* **门控网络 (Gating Network)**：
  * **结构**：轻量级 MLP（如隐层 128 -> 64 -> $K$）。
  * **输出激活**：采用 **Softmax** 函数，输出个股面向 $K$ 个专家的归一化概率权重 $W=[w_1, w_2, ..., w_K]$，满足 $\sum w_i = 1$。
* **专家网络 (Expert Networks)**：
  * **数量**：暂定 $K=5$ 到 $8$ 个平行的小参数量神经网络。
  * **输出**：每个专家基于 $X_{expert}$ 各自预测出当日超额收益信号 $Y = [y_1, y_2, ..., y_K]$。
* **最终加权融合**：
  * $Y_{final} = \sum_{i=1}^{K} (w_i * y_i)$

### 二、 训练流程设计（两阶段联合优化范式）

门控网络的训练目标不是去拟合一个虚构的“专家标签”，而是直接为最终的预测误差 $MSE(Y_{final}, Y_{true})$ 负责。为了防止多专家模型在一开始训练时走向同质化（坍塌），采用**预训练 + 微调**范式：

#### 阶段1：专家初始化预训练 (Warm-up Phase)
**目标**：强制各个专家在初始化时，具备不同的局部市场（子域）感知力，打散同质性。
1. **历史数据聚类**：使用高斯混合模型（GMM）或 K-Means，仅利用 $X_{gate}$（市值、波动平等），将全市场历史日频切片强行分为 $K$ 份子集。
2. **独立训练**：针对这 $K$ 份数据子集，单独预训练 $K$ 个专家网络（拟合真实的超额收益率标签 `future_ret_5d`）。此时，各个专家“术业有专攻”。

#### 阶段2：MoE端到端联合优化 (End-to-End Joint Training Phase)
**目标**：放开人为聚类边界，让门控网络和专家模型通过回传梯度，共同学习“在边界地带如何平稳交接兵权”。
1. **网络拼接**：把阶段1预训练好的 $K$ 个专家模型权重作为初始参数，接上随机初始化的门控网络，组成完整的 MoE 大网。
2. **全样本喂入**：取消任何切片划分，将每日数千只股票的全量 $[X_{gate}, X_{expert}]$ 一并投入网络。
3. **损失函数构造与反向传播**：
   * 采用全量预测误差：$Loss_{pred} = MSE(Y_{final}, Y_{true})$ （或者带有排序性质的 InfoNCE 损失）。
   * **引入负载均衡惩罚（Load Balancing Loss）**：为防止门控网络偷懒（比如永远只把权重1.0分给表现最好的专家A），强加约束，促使总体样本的分配权重方差不过度走高：$Loss_{total} = Loss_{pred} + \alpha \cdot L_{balance}$。
   * **联合梯度更新**：依据 $Loss_{total}$ 的反向传播（Backpropagation），**同时自动更新**所有专家网络和受梯度制导的门控网络权重分布。

### 三、 实时推理及部署落地设计

* **推理时钟**：每日收盘拿到最新数据截面特征，触发前置特征标准化（只需算出全市场均值与标准差）。
* **并行图计算效率**：由于舍弃了为数众多个独立树模型的数据切片传递，推理时数千只个股的矩阵计算可由 GPU 直接**并行执行**所有的专家网络和门控网络点乘。推理延迟预计在百毫秒量级，极为强悍。
* **迭代与更新频率**：考虑到神经网络防过拟合的诉求，可支持：
  * 滚动训练（Rolling Window）：每季度拿过去三年的数据，复核门控边界和专家表征能力。
  * 增量训练（Incremental）：每月将新增的一批月度数据送入全网跑少量 Epochs 进行软微调。

### 四、 消融实验与无Baseline开局指引

考虑到是“从零搭建，没有基线（Baseline）”，在科研与落地工程双轨评估下，须循序渐进跑出可信基线：

#### 对比实验链路规划
| 实验基栈名称 | 网络架构配置 | 验证目标与对照意义 |
| :--- | :--- | :--- |
| **Baseline-0 (单极网)** | 单个参数量巨大的统管全局DNN | 这是真正的零起点基准。测得大一统网络无法兼顾局部风格的短板。 |
| **Baseline-1 (硬聚类断崖)** | 仅走完上述 **阶段1** 的 $K$ 个硬分派独立模型，不用门控。 | 观测因标签跳变带来的持仓换手率极高、甚至滑点导致的收益断层（复现了所谓的“簇漂移痛点”）。 |
| **MoE-Core (本方案终端)** | 本文描述的**阶段1（预训练）+ 阶段2（端到端）**的最终版本。 | 对比 Baseline-1，重点盯防验证**换手率是否断崖式下降、IC时序平稳度是否抬升**。 |

#### 超参数扫描指引
* **专家数量 $(K)$**：建议网格搜索 $K = [4, 6, 8, 12]$。如果过多就会导致负载极度偏斜、训练失效。
* **门控复杂度**：只需控制隐藏层在 1 到 2 层即可，这只起到一个宏观寻优的路由功能。它如果太厚容易发生震荡，喧宾夺主。
