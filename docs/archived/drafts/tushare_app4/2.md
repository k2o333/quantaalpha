  ### Task 3: Replace Python duplicate scanning with Polars while preserving
  primary-key semantics

  Files:

  - Modify: app4/core/processor.py
  - Test: app4/tests/test_processor_polars.py
  - [ ] Step 1: Add failing tests for primary-key-based duplicate detection

  Cover these cases:

  - same primary key, different non-key fields => counted as duplicate
  - identical full rows => counted as duplicate
  - no configured primary key => duplicates count as zero
  - missing subset of primary keys in DataFrame => only existing key columns
    participate
  - [ ] Step 2: Run the processor test file

  Run: python -m pytest app4/tests/test_processor_polars.py -v
  Expected: FAIL until logic is switched from to_dicts() path

  - [ ] Step 3: Refactor _handle_primary_keys() to avoid df.to_dicts()

  Do not use df.is_duplicated() on the whole row.
  Instead:

  - derive existing_keys = [k for k in primary_keys if k in df.columns]
  - if existing_keys is not empty, compute duplicate count by primary key
    only

  Recommended implementation direction:

  duplicate_count = (
      df.group_by(existing_keys)
        .len()
        .filter(pl.col("len") > 1)
        .select((pl.col("len") - 1).sum())
        .item()
  )

  If duplicate_count > 0, log warning.

  - [ ] Step 4: Apply predefined schema by column cast, not DataFrame rebuild

  Replace:

  df = pl.DataFrame(data_list, schema=predefined_schema)

  With:

  cast_exprs = [
      pl.col(col).cast(dtype, strict=False)
      for col, dtype in predefined_schema.items()
      if col in df.columns
  ]
  if cast_exprs:
      df = df.with_columns(cast_exprs)

  This avoids a second materialization and preserves non-schema columns.

  - [ ] Step 5: Refactor validate_data() to use Polars duplicate stats

  Compute stats from DataFrame directly, still based on existing_keys, not
  whole-row duplication.

  - [ ] Step 6: Keep _detect_duplicates_fast() temporarily, but stop calling
    it internally

  Reason:

  - minimize blast radius
  - preserve backward compatibility if another path still imports it

  Mark with a short TODO comment for later removal after repo-wide grep
  confirms no runtime usage.

  - [ ] Step 7: Re-run processor tests

  Run: python -m pytest app4/tests/test_processor_polars.py -v
  Expected: PASS

  - [ ] Step 8: Commit

  git add app4/core/processor.py app4/tests/test_processor_polars.py
  git commit -m "perf: move processor duplicate stats to polars"

  Rollback point: revert this commit if duplicate counts change unexpectedly

  ## Risk Notes

  - Biggest semantic risk is duplicate detection changing from “按主键” to
    “按整行”. V2 explicitly forbids that.
  - Biggest concurrency risk is not ThreadPoolExecutor itself, but
    unsynchronized shared state in StorageManager.save_data().
  - Biggest hidden behavior risk is coverage strategy mismatch between
    executor and coverage manager.
  - Do not remove _detect_duplicates_fast() until a repo-wide usage check
    confirms it is dead.

  ———

  ## Rollback Strategy

  1. If pagination behavior regresses, revert Task 1 commit only.
  2. If duplicate counts or validation behavior change unexpectedly, revert
     Task 2 or Task 3 independently.
  3. If mixed-schema payloads fail more often, revert Task 4.
  4. If concurrent stock updates trigger save anomalies or throttling issues,
     revert Task 7 and keep Task 6.
  5. If necessary, disable stock-level parallelism via config by setting
     update.stock_gap_workers: 1 before code rollback.
