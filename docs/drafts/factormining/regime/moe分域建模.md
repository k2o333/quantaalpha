软分域门控模型（MoE）框架型需求文档  
版本：1.0  
适用场景：A股5000只个股动态量价Alpha策略  
核心目标：彻底解决簇漂移问题，实现预测结果平滑过渡、市场动态适配  

一、项目背景与核心问题  
问题描述   传统方案痛点   本方案解决方案
簇漂移导致模型失效   硬聚类后，股票跨簇跳变 → 模型预测剧烈波动 → 交易成本飙升、回撤放大   用门控网络动态分配权重，消除簇标签依赖，预测结果平滑过渡

静态聚类滞后市场   月度聚类无法捕捉周度风格切换（如TMT→消费轮动）   门控网络实时响应市场变化，无需重训模型

高维因子失效   全市场模型因子有效性平均化（大盘股/小盘股因子逻辑冲突）   专家模型专注同质群体，深度挖掘局部因子有效性

✅ 核心价值：模型预测不再受“股票归属簇”的影响，而是基于个股当前特征与群体的相对位置，实现“市场变，模型不动”。

二、系统架构（关键流程）  
graph LR
A[输入：个股日频数据] --> B(特征工程)
B --> C[门控网络 Gating Network]
B --> D[专家模型 Expert Models]
C --> E[权重分配 w1, w2, ..., wN]
D --> F[专家预测 y1, y2, ..., yN]
E & F --> G[加权融合 y_final = Σ(w_i * y_i)]
G --> H[输出：个股预测收益率]

数据输入规范
数据类型   用途   字段要求   禁止使用
聚类特征（用于门控网络）   门控路由   log_market_cap, volatility_20d, industry, value_growth_score   量价/基本面Alpha因子

预测特征（用于专家模型）   专家模型输入   momentum_5d, financing_flow, bid_ask_spread, ROE_20d（共50-100个）   聚类特征（避免重复）

目标变量   训练标签   future_ret_5d（5日超额收益率）   未来数据（禁止）

⚠️ 红线：聚类特征与预测特征必须物理隔离（不可混用）。

三、关键组件需求说明  

门控网络（Gating Network）  
组件   要求   详细说明
输入   个股聚类特征   4-6个稳定特征（如市值、波动率、行业、风格暴露）

输出   专家权重（Σw_i=1）   每个专家权重在[0,1]区间，表示“个股适配该专家的程度”

模型   轻量级神经网络（MLP）   2层隐藏层，128节点，ReLU激活；不输出预测值

训练   仅用历史数据   每季度更新，输入：[聚类特征] + 目标：专家标签（通过软聚类生成）

关键逻辑   自动学习“特征→专家”映射   例：当市值40% → 专家3权重=0.8

✨ 为什么不用K-Means？  
门控网络不依赖硬聚类，而是直接学习特征与专家的关联，避免簇漂移。

专家模型（Expert Models）  
组件   要求   详细说明
数量   10-20个（N）   按A股典型风格分组（如：高波动成长/低估值价值/小盘科技）

输入   个股预测特征 + 簇内相对特征   例：momentum_5d_rel = (momentum_5d - 簇均值) / 簇标准差

模型   LightGBM（回归/排序）   objective=lambdarank（优化选股排名）

训练   每季度更新   用全量历史数据，但仅训练当前专家的专属样本

特征工程关键   必须包含相对特征   避免输入“簇平均值”（如cluster_avg_momentum），必须用rel特征

💡 相对特征示例：  
全市场平均动量：1.2%  
专家3（高波动成长）群体均值：3.5%  
个股动量：4.2% → momentum_rel = (4.2% - 3.5%) / 2.0% = 0.35  → 该特征表示：个股比群体强35%

加权融合机制  
步骤   逻辑   作用
门控网络输出权重   w1=0.4, w2=0.3, w3=0.3   表示个股适配专家1/2/3的程度

专家模型输出预测   y1=0.02, y2=-0.01, y3=0.03   专家1预测收益2%，专家2-1%，专家3+3%

加权融合   y_final = 0.0.02 + 0.3(-0.01) + 0.3*0.03 = 0.016   平滑过渡：避免因簇跳变导致预测突变

✅ 效果对比：  
硬聚类方案：股票从簇A→簇B → 预测从+2% → -3%（剧烈跳变）  
MoE方案：股票从簇A→簇B → 预测从+2% → +1.6%（平滑过渡）  

四、关键约束与红线（必须遵守）  
红线   说明   违反后果
1. 聚类特征 ≠ 预测特征   聚类特征仅用于门控，预测特征仅用于专家   模型学习“簇标签”而非真实因子规律 → 过拟合

2. 绝对禁止输入簇平均值   专家模型输入必须是rel特征，不能用cluster_avg_pe   专家模型无法区分个股 → 预测失效

3. 门控网络不预测收益   门控网络仅输出权重，不输出收益率   门控网络与专家模型功能混淆 → 逻辑混乱

4. 模型更新频率 ≤ 季度   专家模型每季度更新，门控网络每季度更新   频繁重训 → 过拟合短期噪声

5. 无未来函数   所有聚类、训练、预测仅用调仓日之前数据   回测收益虚高，实盘必崩

七、为什么必须用方案3？（A股场景适配性）  
问题   方案1/2   方案3   为什么方案3更优
市场风格切换速度   月度聚类滞后（如TMT→消费需1个月）   门控网络实时响应（当日风格切换）   A股风格切换平均周期200%，需平滑过渡

高维因子有效性   全市场模型因子平均化   专家模型深度挖掘局部因子（如小盘股的“游资动量”）   A股小盘股因子有效性比大盘股高30%

实盘交易成本   硬聚类导致持仓大幅波动   预测平滑 → 持仓稳定   降低换手率50%+

💎 结论：方案3是高动态量价策略的唯一解，其他方案在A股高频场景下必然失效。
