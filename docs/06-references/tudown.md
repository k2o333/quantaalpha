# TuShare 数据接口使用指南（仅沪深A股）

## 概述
本指南总结了TuShare Pro版的**沪深A股**数据接口，不包含港股、北交所等其他市场数据，涵盖了A股市场从基础信息到高级分析所需的各种数据。根据数据的性质和更新频率，我们将其分为三类：

1.  **静态基础数据**：一次性下载，之后只需偶尔更新。
2.  **历史行情与财务数据**：一次性下载全量历史数据。
3.  **每日增量数据**：每天收盘后需要例行下载的更新数据。

---

## 第一部分：基础信息数据（静态数据，仅沪深A股）

### 1. 股票基本信息 (stock_basic)
*   **用途**：获取沪深A股的股票代码、名称、上市日期、行业、地域等基本信息，是数据系统的“户口本”。
*   **权限**：2000积分起
*   **调用策略**：一次性下载全部。建议使用 `exchange='SSE'` 和 `exchange='SZSE'` 分别获取上交所和深交所数据，确保不包含其他市场。
*   **接口**：`stock_basic`
*   **字段**：ts_code, name, area, industry, market, list_status, list_date, delist_date, etc.

### 2. 交易日历 (trade_cal)
*   **用途**：获取沪深交易所交易日历数据（SSE/SZSE），用于按交易日循环获取数据，自动跳过周末和节假日。
*   **权限**：2000积分
*   **调用策略**：一次性下载全部（历史至未来），使用 `exchange='SSE'` 或 `exchange='SZSE'` 获取沪深市场日历。
*   **接口**：`trade_cal`
*   **字段**：exchange, cal_date, is_open, pretrade_date

### 3. 上市公司基本信息 (stock_company)
*   **用途**：获取沪深上市公司更详细的公司信息（注册资本、法人、员工数等）。
*   **权限**：120积分起
*   **调用策略**：一次性下载，分别按 `exchange='SSE'` 和 `exchange='SZSE'` 分批获取，季度更新。
*   **接口**：`stock_company`
*   **字段**：ts_code, com_name, chairman, manager, reg_capital, province, city, employees, etc.

### 4. ST股票列表 (stock_st)
*   **用途**：获取沪深历史上每天的ST股票列表。
*   **权限**：3000积分起
*   **调用策略**：按日期循环获取历史数据，主要用于监控ST股票变动。
*   **接口**：`stock_st`
*   **字段**：ts_code, name, trade_date, type, type_name

### 5. 股票曾用名 (namechange)
*   **用途**：获取沪深上市公司历史名称变更记录。
*   **权限**：无需积分
*   **调用策略**：按股票或日期范围获取
*   **接口**：`namechange`
*   **字段**：ts_code, name, start_date, end_date, ann_date, change_reason

### 6. IPO新股列表 (new_share)
*   **用途**：获取沪深新股上市列表数据。
*   **权限**：120积分起
*   **接口**：`new_share`
*   **字段**：ts_code, name, ipo_date, issue_date, amount, price, pe, etc.

### 7. 备用基础数据 (bak_basic) 
*   **用途**：获取沪深备用基础列表，包含历史每天股票列表及财务指标（如PE、PB、股东数等）。
*   **权限**：正式权限需要5000积分
*   **接口**：`bak_basic`
*   **字段**：trade_date, ts_code, name, industry, area, pe, pb, eps, etc.

**重要提醒：以下接口与您需求无关，已明确排除**
- **沪深港通股票列表 (stock_hsgt)**：此接口涉及港股通，不在您关注范围内
- **北交所新旧代码对照表 (bse_mapping)**：北交所股票，不在您关注范围内

## 第三部分：行情数据（历史+每日，仅沪深A股）

### 1. A股日线行情 (daily)
*   **用途**：获取沪深A股未复权行情数据（开高低收、成交量、成交额）。
*   **权限**：基础积分（每分钟500次）
*   **调用策略**：按沪深股票代码或日期获取，确保只获取沪深A股数据。
*   **接口**：`daily`
*   **字段**：open, high, low, close, vol, amount, change, pct_chg, etc.

### 2. A股复权行情 (pro_bar)
*   **用途**：获取沪深A股复权行情（前复权、后复权），对回测至关重要。
*   **权限**：基础积分（每分钟500次）
*   **调用策略**：按沪深股票代码和复权类型获取。
*   **接口**：`pro_bar`
*   **参数**：asset='E' (股票), adj='qfq' (前复权), adj='hfq' (后复权), adj=None (未复权)
*   **复权机制**：Tushare是根据设定的end_date参数动态复权，采用分红再投模式
*   **重要**：确保只获取沪深A股（TS代码以.SZ或.SH结尾）

### 3. 每日指标 (daily_basic)
*   **用途**：获取沪深全部股票每日重要的基本面指标（PE、PB、换手率、市值等）。
*   **权限**：2000积分起（5000积分无总量限制）
*   **调用策略**：按日期获取当日沪深全部股票指标。
*   **接口**：`daily_basic`
*   **字段**：pe, pb, ps, turnover_rate, volume_ratio, total_mv, circ_mv, etc.

### 4. 通用行情接口 (pro_bar) - 沪深A股专用
*   **用途**：专门用于获取沪深股票行情数据，支持复权、均线等。
*   **权限**：基础积分
*   **接口**：`pro_bar`
*   **参数**：asset='E' (确保只获取股票), freq='D' (日线), 可设置adj参数进行复权

### 5. 备用行情 (bak_daily)
*   **用途**：获取沪深备用行情，包含特定的行情指标。
*   **权限**：正式权限需要5000积分
*   **接口**：`bak_daily`
*   **字段**：包含涨跌幅、换手率、市盈率、强弱度、活跃度等

### 6. 每日停复牌信息 (suspend_d)
*   **用途**：获取沪深A股每日停复牌信息。
*   **权限**：无特定要求
*   **接口**：`suspend_d`
*   **参数**：可使用 `exchange='SSE'` 和 `exchange='SZSE'` 筛选沪深市场

*   **调用策略**：单次最大可调取5000条数据，可以根据日期和代码循环提取全部数据，数据每日盘后更新
*   **接口**：`moneyflow_ind_ths`
*   **参数**：ts_code(行业代码), trade_date(交易日期), start_date(开始日期), end_date(结束日期)
*   **字段**：交易日期、板块代码、行业名称、领涨股票、收盘指数、指数涨跌幅、公司数量、领涨股涨跌幅、领涨股最新价、流入资金、流出资金、净额等

- 合理规划下载时段，避免在系统繁忙时段集中调用

## 重要注意事项

1. **仅沪深A股**：所有配置和代码必须严格针对沪深A股，TS代码格式为 `XXXXXX.SH` (上交所) 或 `XXXXXX.SZ` (深交所)
2. **排除其他市场**：严格排除港股（.HK）、北交所（.BJ）等其他市场数据
3. **日线数据**：主要使用日线数据，已排除周线、月线、分钟数据
4. **复权处理**：需要获取前复权、后复权和原始价格数据。前复权用于回测确保价格连续性，后复权用于查看历史真实价格，原始价格用于当日交易参考
5. **数据完整性**：财务报表数据完整覆盖沪深A股，包括利润表、资产负债表、现金流量表和财务指标
6. **积分门槛**：积分是访问数据接口的门槛，不是消耗品。达到相应积分要求即可访问对应接口，积分不会因调用API而减少
